# 怎么把它跑起来（新手向 · 一步步来）

这是 **Lucky Ducky（小棉鸭）** 的 uni-app 跨端工程，目前已实现**首页**。
你有两种方式看到它：① 浏览器预览（最快，免微信账号）；② 微信小程序预览。

---

## 环境要求

- **Node.js ≥ 20**、**npm ≥ 10**（见 `package.json` engines）。换电脑到 https://nodejs.org 下载 LTS 版即可。
- ⚠️ **必须在项目根目录里操作**：所有命令都要先 `cd` 进 `luckyducky-next/` 再运行，
  否则会"找不到命令/装错地方"。

## 准备工作（只需一次）

1. 打开「终端」(Terminal)，进入项目目录（**这一步不能省**）：
   ```bash
   cd /Users/sparrow/luckyducky-next
   ```
3. 安装依赖（第一次需要，已经装过就跳过）：
   ```bash
   npm install
   ```

---

## 方式一：浏览器预览（推荐先用这个）

```bash
npm run dev:h5
```
- 启动后浏览器打开 **`http://localhost:5173/`**（若 5173 被占用，会自动换 5174 等，**以终端打印的实际地址为准**）。
  想在真手机上看：电脑手机连同一 WiFi，手机浏览器打开终端里那个 `Network: http://192.168.x.x:5173/` 地址。
- 想看手机效果：在浏览器里按 `F12` 打开开发者工具 → 点左上角的「手机/平板」图标 → 选一个手机型号。
- **改代码会自动刷新**，适合一边学一边看。
- 按 `Ctrl + C` 停止。

## 方式二：微信小程序预览

1. 先生成小程序代码：
   ```bash
   npm run dev:mp-weixin
   ```
   （它会持续运行并监听改动；生成结果在 `packages/miniapp/dist/dev/mp-weixin`。）
2. 下载并安装 **微信开发者工具**：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
3. 打开它 →「导入项目」→ 目录选 `packages/miniapp/dist/dev/mp-weixin` →
   AppID 选「**测试号**」（新手不用注册）→ 导入。
4. 模拟器里就能看到首页。

> 小贴士：`dev:` 是开发模式（边改边看）；`build:` 是打包发布模式（如 `npm run build:mp-weixin`）。完整命令清单（含 `npm run check`）见 `CLAUDE.md` §2，本文只讲新手怎么起步。

## 以后想改东西，去哪儿？（速查）

| 想做的事 | 改哪里 |
|---|---|
| 换主题色 / 字号 / 圆角 | `src/uni.scss`（设计变量都在这） |
| 改首页文案 / 产品 / 评价 / FAQ | `src/data/*.js` |
| 把某个灰占位换成真图片 | 给对应 `<MediaSlot>` 传 `:src`（见 `src/components/MediaSlot.vue` 顶部说明） |
| 以后换成视频/动图 | 只改 `src/components/MediaSlot.vue` 一个文件 |
| 加新页面 | `src/pages/` 下新建 + 在 `src/pages.json` 注册；底部 Tab 在 `src/components/TabBar.vue` |
| 接后端接口 | `src/api/`（已留好空壳）；登录/购物车状态在 `src/store/` |

> 更详细的项目约定见 `CLAUDE.md`（也是给 AI 协作时读的说明书）。

## 常见问题

- **`npm run dev:h5` 报错找不到命令？** 先 `cd` 到项目目录，并先跑过 `npm install`。
- **图标/图片不显示？** 确认 `src/static/` 里有对应文件；小程序端图标用的是 svg 文件 + `<image>`。
- **小程序里某些新写法不生效？** 在 `src/manifest.json` 的 `mp-weixin` 里把基础库调到较新版本。
- **构建时一堆 `legacy-js-api` 警告？** 那是 Sass 的提示，不影响运行，可忽略。
- **改了代码但页面没更新？** 先看终端有没有红色报错；如果卡住，按 `Ctrl + C` 停掉再重新 `npm run dev:h5`。
- **图标显示不出来？** 确认 `src/static/icons/` 里有同名 svg，且组件里 `<Icon name="xxx" />` 的 `name` 不带 `.svg`。

## 换电脑 / 新开对话，怎么接着做（重要）

新会话不会记得之前聊过什么，但项目的「记忆」都写在了文件里。这样接上：

1. **在项目目录里启动会话**：让 Claude 的工作目录是 `luckyducky-next/`
   （这样它会**自动读取** `CLAUDE.md` —— 那是本项目的权威说明）。
2. 直接对它说这段「开场白」（可复制）：
   > 继续 Lucky Ducky 项目（uni-app 微信小程序电商）。请先完整读 `CLAUDE.md` 和 `运行说明.md`，
   > 严格遵守里面的约定；看「项目现状」和「路线图」确认进度，再告诉我下一步建议。
3. `CLAUDE.md` 里有三节专门保证延续性：**项目现状**（做到哪了）、**路线图**（下一步）、**关键决策记录**（为什么这么做）。
   每完成一个里程碑，记得让 Claude **同步更新这三节**，并 `git commit`，下次就能继续无缝接上。

> 一句话：**`CLAUDE.md` 就是这个项目的"记忆"**。它跟着代码走、能进 Git、你我都能看能改——这是跨会话一致性的根本保障。
