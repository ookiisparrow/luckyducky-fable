## 时机三层（总纲）
运营动作按成本与时机分三层，下面五维度各自归入：
- **① 上线前必埋钩子**（能力开通 / 数据结构 / 入口 / 来源埋点——后补极贵）
- **② 上线初期 0→1**（私域沉淀 + 召回）
- **③ 增长期 1→N**（内容 SEO/GEO + 裂变活动）

## 一、SEO / GEO
**核心认知**：微信小程序内容不在公网，传统 Web SEO 不适用。三战场：
- **微信搜一搜（主战场）**：小程序名 / 服务类目 / 关键词定对 + 搜一搜内容收录——微信内免费精准流量。
- **公网 SEO**：用 H5 端做可索引的内容 / 落地页站，承接百度 + 公众号外链，引流回小程序（核心交易仍走小程序，T1 不破）。
- **GEO（让生成式 AI 推荐）**：抓手在站外权威语料——小红书 / 知乎 / B站教程、公众号长文。
- **时机**：名称 / 类目 / 关键词＝① 上线前定（改名有成本）；内容站 / GEO＝③ 增长期（依赖真实内容，非上线阻塞）。

## 二、企业微信 / 私域
价值＝客服 + 私域沉淀（钩织高陪伴、高复购、强社群，特别吃私域）。
- **轻量起步**：小程序内「加企微」活码入口 → 社群 / 上新 / 复购召回。
- **要件**：企微主体 + 认证 + 「客户联系」能力（API secret）。
- **技术坑**：云函数调企微 API 需 IP 白名单，而云函数出口 IP 不固定 → 走已有 HTTP 服务固定出口 / 代理（开发期确认）。
- **与微信客服取舍**：即时答疑用微信客服官方组件（`open-type="contact"`）最轻；私域复购才上企微。建议结合。
- **时机**：客服入口＝② 上线初期必须（见 `待办与债` #9 客服方案）；「加企微」入口越早埋越好。

## 三、运营 bug 收集（与 `待办与债` #23 配套）
#23 管后端可观测性，本条管前端 + 用户侧。上线初期是 bug 高发期，**上线前埋**：
- 用户反馈入口（写 `feedback` 集合）+ 微信 mp 后台「用户反馈」。
- 前端错误自动上报（JS 错误 / 白屏 / 接口失败 → 复用 `trackEvent`/events 集合）。
- 微信小程序助手 App 盯崩溃 / 性能；客服对话回流为 bug 线索。
- 闭环：bug → `调试日志.md`（走 `/systematic-debugging`）。
- **时机**：反馈入口 + 错误上报＝① 上线前（与 #23 合做一条）。

## 四、营销策略（开发期埋钩子）
后补极贵的，开发期就埋（数据结构 / 能力 / 入口 / 埋点）：
- **分享裂变**：`onShareAppMessage` + 分享卡片 + 带参路径追来源（`待办与债` #10）。
- **来源埋点**：`trackEvent` 记 scene / 渠道——营销 ROI 地基（events 已有基础）。
- **订阅消息**：提前申模板（订单 / 发货 / 上新 / 开课）+ 埋授权入口——低成本召回。
- **优惠券 / 促销**：数据结构预留（`待办与债` #6 / 需求 R20）。
- **拼团 / 砍价 / 新人券**：依赖分享 + 订单来源，结构预留。
- **小程序码带参**：海报 / 印刷扫码追渠道（`待办与债` #18）。
- **时机**：埋点 / 能力 / 结构＝① 上线前；活动运营＝上线后。**最贵的三样**：分享能力 + 来源埋点 + 订阅消息模板。

## 五、微信生态红利
已吃微信支付（已通）。可继续吃：
- **流量入口矩阵**：搜一搜 / 附近 / 公众号 / 视频号 / 朋友圈广告 / 扫码 / 分享——获客成本远低于独立 App。
- **视频号 + 直播**：钩织教学卖货天然适配——「制作陪伴」承重柱最大内容杠杆。
- **微信小店**：小程序 / 视频号 / 公众号电商打通的统一带货基建，值得评估。
- **订阅消息 / 服务通知**：免费、高到达的召回通道。
- **⚠️ 合规**：交易类小程序须录入发货信息（发货管理 API），不接影响资金结算 → 确认是否已接官方「发货信息录入」（① 上线前合规）。
