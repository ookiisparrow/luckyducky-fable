# 后台360工作站（自建客服 + 360 + 私域沉淀体系）

> **这是什么**：本文件夹 = 「后台360工作站」子项目的**活文档家**(规范/蓝图/进度)。客服需求经 10 轮爆破 + 场景审核 + 反向审核收敛后，用户 2026-06-30 **确定自建**，单开此文件夹承载建设。
> **不占 docs-budget**：`docs-budget` 守卫只数 `docs/` 顶层 `.md`，本子文件夹不计入 15 份活文档预算。

## 一句话

客服 = 「**微信生态多节点接住 + 统一 360 + 按意图分流**」服务体系：订单类走微信客服(机器人+360 自动答)，钩织学习求助类引导加企微做 VIP 私域沉淀；坐席在工作台看客人的订单/物流/画像/**学习位置/节点成果照片**精准指导。能力主体全自建在自有 CloudBase，碰微信的接缝收口单点。**顶层设计＝组件化/模块化**(见 `架构规范.md`)。

## 关键决策（已拍板·详见指针）

- T1 改写(非推翻·放开微信生态分发) · **360 工作台＝自建**(不采购/不中台/不 fork 开源) · AI＝Hunyuan 押后
- **双轨**(订单→微信客服 / 学习→企微) · **承面 A+C 混合**(内部 VIP=企微侧栏 / 外包订单=自建独立后台) · 沉淀＝「联系我」活码 · 入口话术功能化(不诱导)
- 性价比：特异层(双源360/学习诊断)必自建·标准层按坐席数/年数算·三否决票(数据归属/微信深绑/运维)→ 当下自建；承面 C 那刻重评自建 vs 买 SaaS
