## P0 · 运营前置（靠人·并行办·**只卡承面/自动跳/查订单·不卡 B0/B1.1**）
- **链②**：小程序绑微信开放平台 + 配 `kfBind` env → 测试用户登录(写 `kfIdentity`)→ 微信客服发消息 → `resolveOpenid` 出 openid + `summarizeOrders` 出单(顺解"查订单")。**360 自动跳/查订单 的命门·非 B0/B1.1 前置**。
- **48002**：建自建应用 → 客户联系权限 + 可信 IP。(转人工分流/承面A侧栏/活码 前置)
- **课程类目资质** 核实；**`send_msg` 人工接待会话窗口** 真机验(定承面C设计·外包中期上前须验)。

## B0 · 模块化地基（顶层框架先行·在 B1.1 之前·遵架构规范）
- **provider 接口 + 板块注册表**：定义 `CustomerPanelProvider {key,label,enabled,order,fetch(openid)}` + registry；admin 前端板块注册表。
- **feature-flag registry**：`config.cs.modules.<key>.enabled` 驱动·板块可开关。
- **立 4 守卫**：`cs-module-registered` / `cs-no-cross-module-import` / `cs-panel-via-provider` / `cs-module-toggleable`(先红后绿·反向自检)。
- 🏁 **M0 模块骨架就位**：此后每批作为「模块」按框架接入、核心零改。

## 阶段一 · 只读 360（天级·快速见效·验数据底座）
> ⚠️ **信任边界前置（开工前核出·2026-06-30·改了「零风险」旧表述）**：360 聚合是本项目**第一个「坐席批量读他人全量数据」的越权面**(订单+物流+PII+学习轨迹)。现有 `admin-actions-audited` 因 `shouldAudit` 跳 `^(get|list)`(`kit/audit.ts`)**不覆盖** `getCustomer360`/`getUser` 这类读 → PII 访问 0 留痕。故 **B1 三件套与数据暴露同生·不滞后**：① 360 读审计破例(守卫候选 `cs-360-read-audited`) ② 坐席 RBAC 骨架(守卫候选 `cs-360-rbac-gated`·哪怕先粗粒度·别让单超管裸奔到外包阶段) ③ 数据共享告知同意(候选 `cs-data-share-consented`)。阶段五 B5.2/B5.4 是「细化/扩展」承接 B1 骨架·**非从零**。详 `架构规范.md §1.5`。
- **B1.1 `getCustomer360` 编排器 + 首批 provider**(S)：编排器遍历 registry；首批 provider(订单/物流/激活/售后)复用 `resolveOpenid`/`getOrderDetail`+`activationFor`/`summarizeOrders`/`getTxAlerts`。**按 provider 模式建·不写大 if-else**(架构规范铁律三)。**每 provider 查询须 bounded**(纳入 `capacity-reads-bounded` 扫描面·防大客户 360 拖垮)。守卫:`cs-panel-via-provider` + **`cs-360-read-audited`**(注:`admin-actions-audited` 跳 `^get`、**不覆盖**本 action·须破例留痕)。
- **B1.2 客户检索 + `getUser`**(S-M)：新 action `searchCustomer`(openid/手机/单号/昵称)+ `getUser`(读 `users`·现无)。
- **B1.3 画像 rollup provider**(S-M)：总消费/单数/激活率/最近活跃(从 orders/activations/events 派生)·作为一个 provider。
- **B1.4 「客户360查询页」admin UI**(S-M)：`packages/admin` 新 `Customer360.vue`·经板块注册表渲染 provider 面板·复用 Orders/Refunds 组件。
- 🏁 **M① 坐席搜一个客人→看全貌(只读)**。

- **B3.2 企微侧栏 H5(承面A)**(M)：侧栏 H5 → `getCurExternalContact` → `idconvert` → `getCustomer360`(复用编排器)。前置 48002 + 真机验侧栏取 id。
