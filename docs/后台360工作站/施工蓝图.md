## 阶段一 · 只读 360（天级·快速见效·验数据底座）
> ⚠️ **信任边界前置（开工前核出·2026-06-30·改了「零风险」旧表述）**：360 聚合是本项目**第一个「坐席批量读他人全量数据」的越权面**(订单+物流+PII+学习轨迹)。现有 `admin-actions-audited` 因 `shouldAudit` 跳 `^(get|list)`(`kit/audit.ts`)**不覆盖** `getCustomer360`/`getUser` 这类读 → PII 访问 0 留痕。故 **B1 三件套与数据暴露同生·不滞后**：① 360 读审计破例(守卫候选 `cs-360-read-audited`) ② 坐席 RBAC 骨架(守卫候选 `cs-360-rbac-gated`·哪怕先粗粒度·别让单超管裸奔到外包阶段) ③ 数据共享告知同意(候选 `cs-data-share-consented`)。阶段五 B5.2/B5.4 是「细化/扩展」承接 B1 骨架·**非从零**。详 `架构规范.md §1.5`。
- **B1.1 `getCustomer360` 编排器 + 首批 provider + 模块框架地基**(S-M·并入原 B0)：**随本批长出框架**——`CustomerPanelProvider {key,label,enabled,order,fetch(openid)}` 接口 + provider registry + feature-flag(`config.cs.modules.<key>.enabled`) + **4 模块守卫**(`cs-module-registered`/`cs-no-cross-module-import`/`cs-panel-via-provider`/`cs-module-toggleable`·先红后绿·反向自检)。编排器遍历 registry；首批 provider(订单/激活先行·复用 `resolveOpenid`/`summarizeOrders`/`getOrderDetail`/`activationFor`；物流/售后随后)。**按 provider 模式建·不写大 if-else**(架构规范铁律三)。**每 provider 查询须 bounded**(纳入 `capacity-reads-bounded` 扫描面·防大客户 360 拖垮)。信任边界守卫(§1.5)同批立**两条**：**`cs-360-read-audited`**(注:`admin-actions-audited` 跳 `^get`、**不覆盖**本 action·须破例留痕·FORCE_AUDIT 名单) + **`cs-360-rbac-gated`**(坐席能力位 `customer:view`·`adminConfig.auth.caps`·ACTION_CAPS 闸)。告知同意 `cs-data-share-consented` **挪 B3.3**（第三方访问出现才有此义务·防提前守空气·防过度工程裁决）。
- **B1.2 客户检索 + `getUser`**(S-M)：新 action `searchCustomer`(openid/手机/单号/昵称)+ `getUser`(读 `users`·现无)。
- **B1.3 画像 rollup provider**(S-M)：总消费/单数/激活率/最近活跃(从 orders/activations/events 派生)·作为一个 provider。
- **B1.4 「客户360查询页」admin UI**(S-M)：`packages/admin` 新 `Customer360.vue`·经板块注册表渲染 provider 面板·复用 Orders/Refunds 组件。

## 阶段六 · 承面 C（外包会话工作台·L·最大块·**buy-candidate**·外包真要上时建）

> **开建前置 + 买/建评估（2026-06-30 核出·像 §1.5 钉死·开建/评买前必过）**：
> 1. **send_msg 人工接待窗口** 🔴真机验（命门·决定自建会话收发可不可行·没验不开建）。
> 2. **买 vs 自建重评**（决策 §19 留口）：通用坐席台是成熟 SaaS 商品·自建难 match 打磨/可靠/可用——**先评 SaaS 能否「接微信客服 kf 渠道 + 嵌双源 360」**：能则买（省自建那座山·按坐席数算）·不能则自建最小台。
> 3. **§1.5 越权剩口（外包＝最强越权面·补齐才上外包）**：防批量导出❌无规范 / **审计 agent_id**（B5.4·未做·多账号已上·审计现仍 `operator:'admin'`）/ PIPL 受托处理协议❌（法务·归律师·守 cc-scope 边界）/ 数据共享 consent（B3.3·外包看客户数据的触发·延后的须补）。
> 4. **自建天花板诚实标**：「便捷/可用/完善」无机器守卫·只能真坐席真会话验（根因#8 [靠人]）+ 小步迭代·别过度承诺。
> 5. **不并行**：承面 C 是紧耦合单件（会话+实时+排队+UI 互锁）·一条线串行 + 借开源聊天 UI 减量·别拆多 CC（合并地狱·并行只利独立模块）。
