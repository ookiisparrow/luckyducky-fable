# 微信客服智能客服 配置手册（runbook）

> **用途**：从零把「微信客服 in-chat 智能客服」配通的完整流程 + 排障手册。2026-06-17 与用户真机逐步联调跑通后沉淀（曾踩 4 个大坑、花了不少时间，全记在 §6 排障）。
> **代码侧**：CS 域 `cs/kfCallback`（回调主函数）+ `cs/kfBind`（身份桥接）+ 接缝单源 `kit/wecom.ts`；功能逻辑在 `cs/kfCallback/dispatch.ts`。客观接口/集合事实在 `系统事实.md`，本文只管「怎么配 + 怎么排障」。
> **密钥不入库**：Token/Secret/EncodingAESKey 的**值**只在云函数环境变量里，本文一律只写变量名、不写值。

## 0. 一张图看清架构

```
顾客微信  →  微信客服会话框  →（消息事件回调）→  自有域名 kf.luckyducky.cn/kfcallback
                                                         ↓（HTTP 访问服务）
                                                   云函数 kfCallback
                                                         ↓
                              验签解密 → 拉消息 sync_msg → 关键词识别(dispatch)
                                                         ↓
                              回 msgmenu 气泡 / 文字 / 小程序卡片 / 转人工（send_msg）
```

**关键认知（最容易搞错的一点）**：微信客服的顾客消息**回调到「微信客服独立后台 → 企业内部接入」配的那个回调**（`kf.weixin.qq.com`），**不是**企业微信「自建应用 → 接收消息」那个。后者你也能配、echostr 也能验过，但客服消息根本不会发到它——别在那儿浪费时间。

## 3. 环境变量清单（云函数 kfCallback / kfBind）

| 变量 | 必填 | 说明 | 取值来源 |
|---|---|---|---|
| `WXKF_CORPID` | ✅ | 企业 ID | 微信客服后台 → 开发配置 → 企业ID（`wwda6861818cb50dd9`）|
| `WXKF_SECRET` | ✅ | **微信客服「开发配置」页的 Secret**（不是自建应用的 Secret！见 §6 坑③）| 微信客服后台 → 开发配置 → Secret |
| `WXKF_TOKEN` | ✅ | 回调验签 Token | 自己设，与企业内部接入回调表单填的一致 |
| `WXKF_AESKEY` | ✅ | 回调消息 EncodingAESKey（43 位）| 自己设，与企业内部接入回调表单填的一致 |
| `WXKF_SERVICER` | 转人工需要 | 转人工时转给的接待人员 userid | 微信客服后台 → 接待人员 |
| `WXKF_MINIAPP_APPID` | 卡片需要 | 小程序 appid | 小程序后台 |
| `WXKF_THUMB_MEDIA_ID` | 卡片需要 | 小程序卡片缩略图素材 id | 企业微信素材接口上传得 |

> **Token/AESKey 务必用无歧义字符**（避开大写 I、小写 l、数字 0/1、大写 O）——曾因 I/l 抄串导致 echostr 验不过（§6 坑②）。生成干净值：`node -e` 随机取安全字符集，见 §7。
> **改完 env 立即生效**（云函数下次冷启动读新值）；若改了 `WXKF_SECRET` 且 token 已缓存，需删 `kfState` 集合里 `_id="token"` 那条强制重取（§6 坑③）。

## 4. 数据库集合（必须先建，否则不回复）

云开发控制台 → 数据库 → 新建集合（缺了会导致每条消息被当「已读」跳过，§6 坑④）：

- `kfState`：存 access_token 缓存、sync 游标、消息去重痕（`seen:<msgid>`）。
- `kfIdentity`：存 external_userid→openid 身份映射（查订单用）。

建议两集合「数据权限」设**仅管理端可读写**（云函数自带管理权限不受限，但挡前端越权）。

## 5. 配置步骤（按顺序，env 必须先于点「完成」）

1. **建集合** `kfState`、`kfIdentity`（§4）。
2. **配云函数环境变量**（§3）——至少 CORPID/SECRET/TOKEN/AESKEY 四个，保存。
3. **部署 kfCallback / kfBind**（敏感函数·见 §7 部署命令）。
4. **绑回调路径**：云开发 HTTP 访问服务把 `kf.luckyducky.cn/kfcallback` 路由到 `kfCallback` 函数。
5. **填企业内部接入回调**：`kf.weixin.qq.com` → 开发配置 → 回调配置 → 修改：
   - 回调URL `https://kf.luckyducky.cn/kfcallback`
   - Token = `WXKF_TOKEN` 的值；EncodingAESKey = `WXKF_AESKEY` 的值
   - 点「完成」→ 微信发 echostr 验证 → kfCallback 验签回明文 → 通过即「已接入」。
   - ⚠️ 冷启动可能 4–7 秒、逼近微信 ~5 秒超时；若第一次失败别改东西、立刻**再点一次**（函数已热）。
6. **真机验**：非企业成员的顾客微信进客服会话，发「物流」→ 应弹高亮菜单气泡。

## 6. 排障手册（真机踩过的坑，按现象查）

**现象：点「完成」报「请填写带 http 或 https 协议头的合法 URL」**
→ URL 框开头/结尾混进了空格（开头空格会让它判定不是 https 开头）。清空手动重输、确认无空格。

**坑②｜现象：报「openapi回调地址请求不通过」（URL 格式已对）**
→ 微信真发了 echostr 但函数验不过。99% 是**表单里的 Token/AESKey 与 env 不是同一个字符**（大写 I vs 小写 l vs 0/O 抄串）。**定位法**：用 §7 的脚本本地构造一个真 echostr 请求打线上，看回的是不是明文——回空＝验签/解密对不上。**根治**：用无歧义字符重新生成两个值，表单 + env 从同一份复制粘贴。

**坑③｜现象：echostr 过了、发消息后日志 `[kf] sync errcode:95012`、无回复**
→ `95012 = 无权限调用接口`。`WXKF_SECRET` 填错了——用成了**企业微信自建应用的 Secret**，要用**微信客服「开发配置」页的 Secret**。两者 token 权限范围不同，自建应用 token 调不了 `kf/sync_msg`。改 env 后若旧 token 已缓存，删 `kfState` 的 `_id="token"` 记录强制重取。（实测：微信客服 secret 不卡可信IP，云函数任意出口 IP 可调；自建应用 secret 才要配可信IP，会报 60020。）

**坑④｜现象：sync 成功 `errcode:0` 但日志全是 `[kf] dup-skip`、不回复**
→ `kfState` 集合没建。去重靠往 `kfState` 写「已读痕」，集合不存在→写失败→每条消息被当「已处理过」跳过。建 `kfState`、`kfIdentity` 两集合即可。

**现象：一次冒出好几个气泡**
→ 多为**历史积压一次性回**（之前失败期间发的消息攒着，首次成功 sync 全回了）。去重生效后同一条不会再重复回，发一条新消息验证「一条一回」即可。

**现象：进会话欢迎气泡发不出 `[LD_ALERT] SENDMSG_FAILED errcode:40058`**
→ `40058 = 不合法参数`。常见两因：① 进会话事件的 `external_userid` 在 `event` 子对象、取顶层取成空 touser（已在代码修，抽 `extUserId` 兜两种 wire 形状）；② 缺 `WXKF_MINIAPP_APPID`/`WXKF_THUMB_MEDIA_ID` 致小程序卡片参数不全。

**通用：看链路日志**
`npx tcb fn log kfCallback -e cloudbase-d4gcssqbv06865479`，盯 `[kf]` 全链路（onEvent / cursor-read / sync{errcode,msgs,nc} / handle / sent / dup-skip）+ `[LD_ALERT]` 告警码。

## 7. 实操工具

**部署 kfCallback**（敏感函数·guard-deploy 会二次确认）：
```bash
npm run build:cloud
npx tcb fn deploy kfCallback --dir packages/cloud/dist/kfCallback --force -e cloudbase-d4gcssqbv06865479
```

**生成无歧义 Token + EncodingAESKey**：
```bash
node -e 'const c=require("crypto");const BAD=new Set(["I","l","O","0","1","o","i"]);const S="ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz23456789".split("").filter(x=>!BAD.has(x));let t="";while(t.length<16)t+=S[c.randomBytes(1)[0]%S.length];let a;do{a="";while(a.length<43)a+=S[c.randomBytes(1)[0]%S.length]}while(Buffer.from(a+"=","base64").length!==32);console.log("Token =",t,"\nAESKey=",a)'
```

**本地构造真 echostr 验证请求打线上**（定位坑②/坑③，传 Token、AESKey）：思路＝按 WXBizMsgCrypt 用 AESKey 加密一段明文得 echostr、用 Token 算 msg_signature、GET 打 `kfcallback`，看 body 是否回明文。完整脚本见 2026-06-17 会话记录（`kftest`）。

## 8. 功能清单（机器人能干啥）

逻辑全在 `cs/kfCallback/dispatch.ts`：
- **进会话欢迎**：顾客打开对话 → 弹 5 大入口菜单。
- **关键词识别**：4 大类（物流 / 激活码·课程·视频 / 退换·售后 / 钩织教程）→ 弹对应高亮 msgmenu 气泡。
- **菜单点选分流**：下级菜单 / FAQ 文字答 / 小程序卡片 / 查订单 / 转人工。
- **FAQ 文字答**（写死 4 条）：多久发货、改地址、激活码怎么用、退换政策。
- **查我的订单**（实时·零幻觉）：经身份桥接查最近 3 单状态+运单（需小程序绑开放平台 + 顾客登录绑定，暂不可用）。
- **小程序卡片**：申请售后页、我的课程页（需 §3 卡片相关 env）。
- **转人工**：点「人工」或涉承诺/纠纷 → 转接待人员（需 `WXKF_SERVICER`）。
- **去重 / 防伪**：同一条只回一次；回调验签解密 fail-closed。

**改话术/关键词/菜单/FAQ**：改 `dispatch.ts` → `npm run check` → 重新部署 kfCallback。目前没有可视化后台改话术，若需要是另一块要单独做的活。
