# 微信客服 in-chat 智能客服 — 后端交接规格

> 来源：2026-06-16 用户 × 前端 CC（parallel-b）高密度共创设计**定稿**，**交后端 CC（`packages/cloud`）落地**。完整决策来龙去脉见 plan `~/.claude/plans/b-optimized-quill.md`。本文 = 自洽执行简报。
> **落地后本文卷档 archive**（docs-budget·一需求一家·这是临时任务规格非长期活文档）。

## 目标
微信客服「点客服 → 进对话框 → **关键词识别 + 高亮 `msgmenu` 菜单气泡分流** → **同一对话**转人工」统一智能客服。
**避免幻觉第一**：结构化用**确定性 msgmenu + 查实时数据**，**非生成式 AI**（AI/RAG 仅长尾兜底·本期后置）。统一场景 = 帮助中心与人工在同一对话、不分开（用户拍板）。

## 前置（靠人·用户做）
- 企业微信**企业认证**；微信客服「开发配置」拿**微信客服 Secret** + 配回调 URL/Token/EncodingAESKey（指向 `kfCallback`）。
- 小程序**绑微信开放平台**（主体已确认与企业微信一致 → 身份桥接可行）。

## 后端组件（`packages/cloud`·贴现有 kit 模式）
1. **`kfCallback` 云函数 + HTTP 访问服务路径**（类比 `adminApi` 的 `/adminapi`）：GET 验 URL(echostr) / POST 收事件；企业微信**验签(msg_signature) + AES 解密**(WXBizMsgCrypt)·**fail-closed**（病根#3·带异常身份即静默 ACK 不给探测信号·类比 `kit/notify` defineNotifyCallback）。
2. **access_token 管理**：微信客服 Secret→gettoken·缓存 7200s（DB·类比 `kit/throttle` 的 DB 模式）。
3. **`sync_msg` 拉消息**：cursor 持久化 + msgid 去重。
4. **分流引擎**：**关键词识别**（退换/物流/激活/课程…）→ 发 `msgmenu` 气泡（head **高亮识别词** + list **高亮可点选项** + tail「或直接打字」）；按回传 `menu_id` → 下级菜单 / `miniprogram` 小程序卡片 / 文字答案 / 转人工。
5. **身份桥接（查「你的订单」）**：`login` 存 unionid + 调企业微信 `unionid↔external_userid` 转换 API + 建 `external_userid↔openid` 映射表 → 命中订单类查现有 `getMyOrders` 等 → 文字回会话。限制：转换 API 限频(5万/时·24万/天·禁批量·须用户主动触发)、微信客服 external_userid 仅返 **48h 内**有会话的、非客户返 `pending_id`(90 天)。
6. **转人工**：超范围/拿不准/承诺（退款金额/特批/赔偿）→ 同会话转接待人员（企业微信成员）。AI/机器人不拍板。
7. **密钥 env**：corpid / 微信客服 secret / callback token / EncodingAESKey（云函数环境变量·类比 `ADMIN_BOOTSTRAP_KEY`）。
8. **守卫/正册**：新敏感函数进 `scripts/guard-deploy.mjs` SENSITIVE_FNS + 接口正册登记（`interface-catalog-sync` 会拦）+ 回调验签写**反向自检单测**（病根#3 fail-closed）。

## 消息类型（微信客服发送 API）
`msgmenu`（菜单·head/list/tail·点回 `menu_id`·移动端 3.1.12+）/ `miniprogram`（小程序卡片·跳现有页）/ `link` 图文 / text。

## 前端（parallel-b）配合
- `login` 拿/存 unionid（小程序登录链路 ↔ 后端 code2session）——到 P2 我对接，不阻塞。
- 可能的小程序卡片落地页。**不建小程序帮助中心页**（重心 = in-chat 统一）。

## 不在本期
- **L3 RAG AI 长尾**（关键词没命中再议·国产模型 + RAG·避免幻觉走检索增强 + 兜底转人工）。
- **L4 钩织陪伴 / 多模态**（产品战略 track，非纯客服工程）。

## 验证（根因#8 真机）
真机点客服 → 收 `msgmenu` 气泡 → 点选项 → 收对应下级/小程序卡片/文字 → 转人工进**同会话**；回调验签单测 + 反向自检。
