# 进销存 ERP · 施工蓝图（living·按 `/refactor-batch` 拆批）

> **这是什么**：进销存线的**可执行施工路标**。每批=一个 `/refactor-batch`（先守卫红→改到绿→`npm run check`→反向自检→squash）。量级 S(1-3天)/M(数天~1周)·**看码估·非实测（根因#8）**。需求定稿与进度见 `README.md`（单源）；多 CC 并行机制见 `../后台360工作站/并行工作流.md`（通用·不重抄）。
> **总设计立场**：① 全走 **adminApi actions**（不新增云函数——RBAC 默认拒/操作审计/HTTP 闸全白捡·部署面=重部 adminApi 一个）；② 原料账镜像成品账的**单点收口**范式（`kit/inventory.ts`＝先例）；③ 状态机进 **P3 声明表**（learning 先例「加一张表+扩一目录·近零新机制」）；④ 非耦合模块化＝**接口五门收束**（§3）+ 车道文件级隔离（§4）——后期穿插/增减模块只动自己车道的文件。

## §1 领域模型（集合 7 张·**全部 SCM-0 预登记** `kit/collections.ts`+initDb·控制台锁仅管理端）

| 集合 | 键 | 要点 |
|---|---|---|
| `materials` | `_id`=料号（确定性·见下） | name/category/uom(`count`\|`gram`·**建档锁死**)/supplierId/stock(整数)/threshold(安全库存·缺料预警)/active。**库存直存主档文档**·CAS 条件写（同 inventory 范式） |
| `suppliers` | 自动 id | type(`factory` 厂家\|`outworker` 织女·**织女每人一档**)/name/contact/note/active |
| `purchaseOrders` | 自动 id | supplierId + lines[{materialId,qty,unitPriceFen}] + totalFen + **状态机** draft→ordered→received（+draft/ordered→cancelled）。received **首次流转绑入库副作用**（门1·幂等） |
| `outworkOrders` | 自动 id | workerId + issueLines[最大团原团·按色] + receiveLines[带结·同色] + pieceRateFen/payableFen(=收回数×单价) + **状态机** draft→issued→delivered→settled（+draft→cancelled）。issued 后异常（织女退出/丢料）走**调整单兜底**·MVP 不做逆向流转 |
| `assemblyOrders` | 自动 id | productId/spec/sets + **bomSnapshot（执行时冻结快照·不回读模板）** + consumedLines。**单步执行**（建即执行·无状态机·撤销走调整单）——别为不存在的草稿需求建状态机（CLAUDE §7） |
| `stockLedger` | `_id`=`<docType>:<docId>:<itemKey>`（**确定性=幂等**·撞 id 即并发方已写） | append-only 流水：itemKey(料号 或 成品 `fg:<productId>__<spec>`)/delta(±整数)/docType∈{purchase_in, outwork_issue, outwork_receive, assembly_out, assembly_in, adjust, ship}/at/operator。「如实核销」靠它可查可对 |
| `bomProfiles` | `_id`=productId | 每产品三填空：yarnColors{L,M,S} + packagingMaterialId + cardMaterialId |

- **料号命名（确定性·人可读·＝门3 依赖的契约）**：毛线 `yarn:<color>:<L|M|S>:<raw|knotted>`（knotted 仅 L 档）；产品专属件 `pkg:<productId>` / `card:<productId>`；共用辅料语义名（`stuffing` 填充棉/`hook` 钩针/`marker` 记号扣/`needle` 缝针/`keyring` 钥匙扣/`eyes` 眼睛）。建档程序生成、不手拼。
- **BOM 模板**：存现有 `config` 集合确定性 `_id='scmBomTemplate'`（复用配置单源模式·不立新集合）：`commonLines[{materialId,qtyPerSet}]`（共用料·全产品一致）+ `yarnSlots[{tier,form,qtyPerSet}]`（L=knotted / M·S=raw）+ perProduct 槽（包装/卡片 各×1·料号从 bomProfiles 取）。模板数值（a/b/c 团数、克数）＝业务数据，admin 填、不写死代码。
- **计量**：金额一律 Fen（现有品牌类型·工钱/采购价自动纳钱链守卫）；克/件一律**正整数**（validate 层强制）。**不立 Gram 品牌类型**——uom 是数据驱动（同一行字段按主档定克或件），类型层分不开，硬上=过度工程（§7）。
- **状态机 → P3**：`packages/shared/src/scm.spec.ts` 声明 purchaseOrders/outworkOrders 两张流转表 → `scripts/gen-order-domain.mjs` 多域生成（类型/常量/机读 JSON）→ 守卫 `order-transitions-declared` 扩扫 `functions/admin/adminApi/actions/scm*.ts`。

## §2 业务动线（端到端）

```
采购单 received ──purchase_in──▶ 原料账（materials.stock）
最大团原团(按色) ──issued:outwork_issue──▶ 织女在制 ──delivered:outwork_receive──▶ 带结团入库 + payableFen 应付 ──▶ settled 销账
组装单（产品×N套） ──assembly_out（按 bomSnapshot 扣 共用料+三色毛线[L带结/M·S原]+专属包装/卡片）──▶ assembly_in ──▶ 成品账（inventory·现有）
订单 createOrder 下单即扣（现有 reserveStock·防超卖·热路径不加写） ──▶ shipOrder 首次 shipped 流转 ──ship 流水──▶ 核销留痕
备货计算器（只读）：各产品目标套数 × resolveBom → 对比 stock → 采购缺口（按供应商分列）+ 外协缺口（L 原团→带结 应送数）
```

**诚实边界**：下单预留**不写** stockLedger（钱链热路径不加写·orders 集合本身就是那笔账的单据）；成品流水只有 assembly_in / ship / adjust 三种。对账公式＝Σ成品流水 + orders 预留/回补 ⇄ inventory.stock（盘点时核）；原料侧 Σ流水 ⇄ materials.stock（守卫/测试可锁）。

## §3 接口收束（五门·**全部 SCM-0 定义**·车道只经门不越门——多 CC 并行与后期增删模块的前提）

| 门 | 定义 | 收束的是什么 |
|---|---|---|
| **门1 原料账** | `kit/scmStock.ts`：`applyStockMoves(moves[], doc)`＝CAS 改 materials.stock + 写确定性流水（幂等）。**全库唯一**读写 materials.stock/stockLedger 的地方 | 镜像 stock-atomic-conditional 先例·任何单据入出库都经这一个函数·换存储/加损耗逻辑只改这点 |
| **门2 状态机** | `scm.spec.ts` 声明表 + 现有 `kit/transition.ts`。一次性副作用（入库/发料/应付）**绑首次流转** | 私自越流转/写未声明状态→守卫红·并发重放天然幂等 |
| **门3 配方** | `packages/shared/src/scmBom.ts`：`resolveBom(template, profile, sets)` **纯函数** → `[{materialId, qty}]`（含按料号命名契约推导毛线料号） | 组装(C)与备货计算器(D)共用同一解析·配方语义只此一处·可单测 |
| **门4 成品账** | `kit/inventory.ts` 扩一个 `produceStock(productId, spec, qty)`（CAS 增·无文档则建）——现有文件**唯一**新增出口·SCM-0 落 | 车道 C 只调用、不自己碰 inventory·防超卖语义不被绕 |
| **门5 action 面** | 每车道一个 `actions/scm<Lane>.ts` 文件（**文件级隔离·零并行冲突**）；ACTIONS/系统事实/collections/nav 四登记面 master 串行 | RBAC：**不登记 ACTION_CAPS→默认拒 `admin:write`＝仅超管**（外包坐席天然无权）。真出现仓管专号再立 `scm:manage` cap（§7·现在不立） |

## §4 车道拆分（多 CC 并行·机制引 `../后台360工作站/并行工作流.md`·此处只定内容）

| 车道 | 交付 | 依赖 | 量级 |
|---|---|---|---|
| 🏗️ **SCM-0 地基（master 串行·先）** | 7 集合登记+initDb · 五门 · `scm.spec.ts`+gen 扩域 · 物料/供应商主档 CRUD（`actions/scmMaterials.ts`）· admin 物料页（含**期初盘点/调整**·docType adjust·经门1·须填原因·审计白捡）· admin nav「供应链」组 | 无 | M |
| **A 采购线** | `actions/scmPurchase.ts`（save/markOrdered/receive/cancel/list）· received 绑入库（门1+门2）· admin 采购页 | SCM-0 | S-M |
| **B 外协线** | `actions/scmOutwork.ts`（save/issue/receive/settle/cancel/list）· issue 扣原团 / receive 入带结+定格损耗与应付（门1+门2）· admin 外协页（含织女在制/应付一览） | SCM-0 | M |
| **C 配方组装线** | `actions/scmBom.ts`+`actions/scmAssembly.ts` · 模板/bomProfiles 管理 · 组装执行（门3 解析→快照→门1 扣料+门4 入成品）· admin 配方页/打包页 | SCM-0（门3/门4 已在 0 定义） | M |
| **D 计划核销线** | `actions/scmPlanner.ts` 备货计算器（只读·门3+库存读·缺口按供应商分列）+ admin 备货页 · `shipOrder` 加核销流水（改 `actions/orders.ts`·绑首次 shipped 流转·幂等 `ship:<orderId>:<lineKey>`）· 激活码对账（可选·卡片 stock ↔ qrcodes unused） | SCM-0 | S-M |

- **零共享面设计**：集合已在 SCM-0 预登记（最大冲突源消灭）；各车道集合/action 文件/admin 页面互不重叠；四登记面（ACTIONS/系统事实/守卫表/README 进度）车道**报**、master 整合时统一登记（并行工作流 §5）。`actions/orders.ts` 只有车道 D 碰（无撞车）。
- **波次**：SCM-0 → 第一波 A‖B → 第二波 C‖D → SCM-Z。默认 master+2 车道（同 CS 线验证规模）；A–D 互不依赖，波次只是节奏建议、可按人手/优先级穿插增减。
- **就绪信号**：车道 check 绿 → push `feat/scm-<lane>` → tag `scm/<lane>/ready-<seq>`（机制同并行工作流 §4·前缀换 scm）。

## §5 守卫候选（每批先红后绿+反向自检·roots 归因·登注册表时核 guard-coverage）

| 守卫 | 批 | 不变量 |
|---|---|---|
| `material-stock-single-seam` | SCM-0 | 全库仅 `kit/scmStock.ts` 读写 materials.stock/stockLedger（镜像 stock-atomic-conditional·roots #1/#2） |
| `scm-ledger-idempotent` | SCM-0 | 流水写必确定性 `_id`（roots #2） |
| `scm-uom-integer` | SCM-0 | 单据行数量正整数 + 与主档 uom 一致（validate+测试·roots #8） |
| （扩面）`order-transitions-declared` | SCM-0 | scm.spec 两张流转表纳对账·扩扫 actions/scm*（roots #2·learning 先例·零新守卫只扩面） |
| `bom-snapshot-frozen` | SCM-C | 组装执行只读快照——改模板后重放旧单结果不变（测试锁·历史快照不追新·同订单快照原则） |
| `ship-verify-ledger` | SCM-D | 发货首次流转必落 ship 核销流水（测试·绑 transition 幂等·roots #2） |

**白捡（现有守卫自动覆盖·不用做）**：known-collections-only（7 集合不登记即红）· writes-need-gate（新写库必过闸）· admin-actions-audited（写类 action 自动留痕）· agent-rbac-gated（未登记 cap 默认拒）· interface-catalog-sync（新 action 须登系统事实）· admin-nav-route-synced（新页防死链/孤儿页）· fen-*（工钱/采购价金额链）。

## §6 靠人项（机器守不了·随批清·状态记 README 进度表）

- 控制台锁 7 新集合「仅管理端」（SCM-0 部署时·同 initDb 单源化先例）。
- **期初建账**：现有实物盘点后经「期初盘点」录入（不录=账实必偏·上线首日必做）。
- 真单真机验（根因#8「构建过≠真能用」）：一单真采购→入库→发料→收带结→打包→下单→发货，全链流水对得上才算通。
- 印刷卡片↔激活码批次的使用纪律（先生成码批次再下印刷单·对账页只核不管流程）。
