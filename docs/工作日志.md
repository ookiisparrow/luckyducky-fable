## 2026-06-07 · 商品数据上云(P0 续)

接着 v0.1,把「商品」作为第一条业务数据端到端上云,作为课程 / 订单的样板。

### 做了什么
1. **两个云函数**:`seedProducts`(幂等灌 5 个商品,用业务 id 作 `_id` + `doc().set` upsert)、`getProducts`(按 sort 返回列表)。
2. **数据层收口**:`utils/cloud.js` 加 `callCloud`(集中 `wx.cloud.callFunction`,`wx` 守卫 + 条件编译);`api/shop.js` 的 `getProducts` 接 `callCloud`,H5 / App 回退本地 `catalog`;新建 `store/products.js`(商品 store,不持久化,页面从它取)。
3. **页面接入**:首页 `FeatureProducts` 用 `store.featured`(映射回字符串价喂 `ProductCard`);详情头部 `await store.load()` → `getById` → 本地兜底;`App.vue` onLaunch 加 `load()`。删死代码 `data/products.js`。
4. **部署 + 验证**:补 `cloudbaserc.json`;`tcb fn deploy --all`(自动云端装依赖)+ `tcb fn invoke seedProducts`(写 5 条)+ invoke `getProducts`(读回 5 条,CLI 层面证明数据在云端)。
5. **安全边界**:直接改 live 库做「改价铁证」被 auto 模式安全策略拦(写生产库 / AI 自助加权限规则均拒);改用 Console 无回退警告 + CLI invoke 验证,改价由用户自行执行。

### 产出
- 新增:`cloudfunctions/seedProducts`、`cloudfunctions/getProducts`、`store/products.js`、`cloudbaserc.json`。
- 改:`utils/cloud.js`、`api/shop.js`、`App.vue`、`FeatureProducts.vue`、`detail/index.vue`;删:`data/products.js`。
- 健康:lint ✓ / test 16 ✓ / build 双端 ✓。

### 下一步
- 真机眼校:首页 / 详情商品来自云端(看 Console 无回退警告)。
- 课程 / 订单照此三件套模式上云。
