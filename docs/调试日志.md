## C（fable）· 2026-07-05 · recordAnomaly 真机不落库（部署 inspect 逼出·根因#8·已修+守卫·结案）
**现象**：巡检机 inspect 部署到生产、`tcb fn invoke` 回 `red:["stuck-order"]`，但 `anomalies` 集合恒空（`inspectRuns` 却有 2 条）。**根因（#8 构建过≠真机能用）**：`recordAnomaly` 用 `doc().update()`-first，真机 wx-server-sdk 对缺失文档/未建集合 update **抛错**（非返 updated:0），被外层 fail-soft `catch` 静默吞、再到不了 add fallback——`inspectRuns` 有 add+createCollection 兜底故正常，`anomalies` 无、恒空；**整个 bug 收集器半边（含 notifyAlert 桥）真机不落库**。**为何测试没抓**：桩 `doc().update()` 缺失文档返 updated:0 + `add` 自动建集合，双重掩过真机（桩≠真机经典）。**修**：`recordAnomaly` 改 add-first + createCollection 兜底（不依赖 update-on-missing）。**守卫（治本·让桩诚实）**：桩加 `control.markUncreated` opt-in 逼真机「未建集合 add/update 抛错」+ anomaly.test「集合尚未建也能落库」用例（旧写法必红）。**验证**：重部署 inspect + 重调 → anomalies 落库成功（stuck-order 带 9 单 samples）；check 1386→1387 全绿·反向自检禁 createCollection→用例红→还原绿。**结案**（详 `重构日志.md 2026-07-05 部署记账`）。
