## 已结案（教训沉淀,详细推理见 git 历史）

### 2026-06-07 · 购物车列表显示空白行　✅ 已系统性处理
- **现象**:storage 历史脏数据被 `persist.js` 原样 `$patch` 回灌,购物车商品行字段全空。
- **教训**:**持久化回灌必须过数据契约校验**(`$patch(JSON.parse(...))` 只防解析失败,不防内容残缺;数组类状态最脆弱)。
- **状态**:✅ 已处理——`persist.sanitize` + `keepValid` 契约(cart/address/user)+ 全局错误捕获 + `tests/persist.test.js` 锁定。

### 2026-06-07 · 购物车契约漏校验 qty（外部审核 P2）　✅ 已修
- **现象**:`sanitizeCart` 漏校验 qty,无 qty 的脏条目过清洗 → `count` 累加成 NaN。
- **教训**:**契约要覆盖模板 / getter 实际读取的所有字段**,测试数据要贴近真实结构;布尔字段也要归一化。
- **状态**:✅ 已修——qty 正整数校验 + selected 归一化 + `persist.test.js` 断言锁定。

### 2026-06-07 · 云函数 login 漏写 _openid（P0 登录联调）　✅ 已修
- **现象**:云函数 `add` 的 `users` 记录没 `_openid`,按 `_openid` 查不回。
- **教训**:**云函数是管理端,写库必显式写 `_openid`**(只有小程序端 add 自动写;将来 activations/orders 同理);返回别依赖"add 后立即查",直接构造返回。
- **状态**:✅ 已修——`login` 显式 `_openid: OPENID`。

## 待修（已记录,排期单独处理）

### A · 沉浸页顶部悬浮按钮过高,撞状态栏 + 胶囊　🟡 已修,待真机眼校
- **现象**:商品详情「分享」、课程目录「收藏」、播放页「更多」等浮在图/视频上的按钮 top 太靠上,
  撞状态栏、撞微信胶囊。
- **根因**:浮层按钮 top 用固定 `env(safe-area-inset-top)`+小值,既没可靠避状态栏、也没给胶囊让位。
- **修法**(2026-06-10):`systemBar.js` 新增 `getSystemBarVars()`(CSS 变量形式,CoNavBar 同步复用);
  detail / catalog / welcome / player 四页浮层改为「与胶囊同水平带垂直居中 + 右端 `--gap` 横向让位」,
  条件编译保 H5/App 仍走 `env()` 兜底。
- **状态**:🟡 代码已修,需微信开发者工具 / 真机眼校后关账。

### B · 视频播放页顶部下移避胶囊后露大面积黑块　🟡 随 A 已处理,待真机眼校
- **现象**:播放页顶部 UI 整体下移避胶囊后,顶部露出大片黑块。
- **根因**(已定位):之前的"整体下移"思路把视频容器一起推下来,顶部露出 `.vp` 的黑底。
- **修法**(2026-06-10):视频保持 `inset:0` 铺满到顶,**只给 `.vp-top` 浮层加状态栏 padding**——
  布局上不存在"被推下来的顶部空带",黑块成因消除。
- **同类隐患**:沉浸页避让一律"动浮层、不动底层媒体/背景"。
- **状态**:🟡 代码已修,需真机确认同层渲染下表现一致。

### D · 开发者工具里调云函数触发 WAServiceMainContext timeout　⏳ 待查

- **应急(已撤销)**:2026-06-07 曾在工作区临时把 `utils/cloud.js` 改成 `CLOUD_ENABLED=false` 全禁云、页面回退本地数据;
  这批改动**未提交、已 `git restore` 撤销**——只是把"临时关云"的应对去掉,**timeout 本身根因未查**。
- **根因**:待查。候选方向(未验证):工具版本/基础库版本、`wx.cloud.init` 时机、网络/环境配置、模拟器与真机差异
  (真机上 `getOpenId`/`getProducts` 此前 invoke 是 ok 的,疑似集中在开发者工具模拟器)。
- **同类隐患**:所有走 `callCloud` 的页面在开发者工具里都会受影响;别因工具端 timeout 误判云函数本身有问题。
- **是否结构性**:待定(可能是工具/环境层,非业务代码)。
- **状态**:⏳ 待查。下次开微信开发者工具时复现 + 看控制台/网络面板定位,需用户本机操作。

### C · 订单无实体数据贯通(外部审核 P1,已决定暂缓)　⏳ 待办
