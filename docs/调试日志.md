## 已结案（教训沉淀,详细推理见 git 历史）

### 2026-06-07 · 购物车列表显示空白行　✅ 已系统性处理
- **现象**:storage 历史脏数据被 `persist.js` 原样 `$patch` 回灌,购物车商品行字段全空。
- **教训**:**持久化回灌必须过数据契约校验**(`$patch(JSON.parse(...))` 只防解析失败,不防内容残缺;数组类状态最脆弱)。
- **状态**:✅ 已处理——`persist.sanitize` + `keepValid` 契约(cart/address/user)+ 全局错误捕获 + `tests/persist.test.js` 锁定。

### 2026-06-07 · 购物车契约漏校验 qty（外部审核 P2）　✅ 已修
- **现象**:`sanitizeCart` 漏校验 qty,无 qty 的脏条目过清洗 → `count` 累加成 NaN。
- **教训**:**契约要覆盖模板 / getter 实际读取的所有字段**,测试数据要贴近真实结构;布尔字段也要归一化。
- **状态**:✅ 已修——qty 正整数校验 + selected 归一化 + `persist.test.js` 断言锁定。

### 2026-06-07 · 云函数 login 漏写 _openid（P0 登录联调）　✅ 已修
- **现象**:云函数 `add` 的 `users` 记录没 `_openid`,按 `_openid` 查不回。
- **教训**:**云函数是管理端,写库必显式写 `_openid`**(只有小程序端 add 自动写;将来 activations/orders 同理);返回别依赖"add 后立即查",直接构造返回。
- **状态**:✅ 已修——`login` 显式 `_openid: OPENID`。

### 2026-06-10 · 沉浸页浮层撞状态栏/胶囊 + 播放页避让露黑块（原待修 A/B）　✅ 已修并眼校关账
- **现象**:详情「分享」、目录「收藏」、欢迎「关闭」、播放「更多」等浮层按钮撞状态栏/胶囊(A);
  播放页整体下移避让后顶部露大片黑块(B)。
- **教训**:**沉浸页避让一律"动浮层、不动底层媒体/背景"**;顶部定位别只靠 `env(safe-area-inset-top)`,
  用 `systemBar.getSystemBarVars()` 与胶囊同水平带垂直居中 + 横向让位。
- **状态**:✅ 已修(PR #5)+ 自动化眼校通过(2026-06-10,`scripts/visual-check.cjs`:
  四沉浸页按钮矩形 vs 胶囊/状态栏判定全 ✓、与胶囊中线差 0px,截图复核无黑块;
  真机同层渲染如有差异再开新单)。

### 2026-06-10 · 开发者工具假性「component not found」+ 自动化 RPC 卡死（工具坑,非业务）　✅ 已结案
- **现象**:① 工具报 `pages/checkout/index.json` 引用的页内组件 not found,但 dist 里 8 个产物文件齐全、路径正确;
  ② 自动化 `connect` 握手成功但所有 RPC(systemInfo 等)永不回包,visual-check 卡死。
- **教训**:① **build 重写 dist 期间工具会读到中间状态**(json 先落盘、组件文件后落盘)→ 报假性缺件,点「编译」即消;
  跑自动化眼校期间不要执行 `build:mp-weixin`。② **connect 成功 ≠ RPC 可用**:中途 `cli open` 重开项目会把
  自动化端口留成僵尸;解法 `cli quit` 彻底退出再 `cli auto --auto-port 9420` 重启(已在脚本加 RPC 探活守卫)。
- **状态**:✅ 已结案——脚本注释 + 探活守卫沉淀;重启后四沉浸页眼校全 ✓。

### C · 订单无实体数据贯通(外部审核 P1)　✅ 已修(2026-06-10 订单上云,PR #25)
- **现象**:提交 → 支付成功 → 订单详情不是同一笔数据(订单号随机、跳静态样例页;checkout 只传 amount)。
- **修法**:直接以「订单上云」解决——`createOrder` 云函数(云端定价 + 快照)+ `store/orders.js` 收口,
  checkout → paysuccess → order 详情按订单 id 读同一笔;原「前端快照」方案降级为 H5/App 回退。
- **教训**:贯通型问题若已临近对应的云改造里程碑,直接在云改造里一并解决,避免先做一版前端临时方案再推翻。
- **状态**:✅ 已修;真机端到端按 `验收手册.md` 验收单 E 验证。
