## Context
现状是纯 C 端 demo:登录是空壳(只占 `token`)、课程写死一门(`data/course.js`)、欢迎页只是
"看过"标记、`api/` 是空封装。目标:长出完整业务——**实物购买 → 收货扫二维码 → 激活课程绑定
账户 → 确认进入即失去退货权 → 只能看自己买的课**,并配套管理后台。这套全部依赖后端 + 数据库,
是从纯前端到全栈的跃迁。

## 一、端到端业务流程
微信授权登录(登记账户)→ 买**实物**材料包(微信支付)→ 收货、内含**二维码** →
微信扫码进小程序 → 欢迎页 → **弹窗告知「确认进入即失去退货权」→ 用户确认** →
课程激活并绑定账户(记确认时间戳)→ 课程目录/播放,**只显示自己激活的课**。

## 二、系统架构（三端 + 云开发地基）
| 端 | 职责 | 现状 |
|---|---|---|
| C 端小程序(买家+学员) | 登录 / 买货 / 扫码激活 / 看自己的课 | 有 UI 壳,业务全空 |
| 管理后台(Web) | 课程内容、**二维码批次(生成/核销)**、订单、用户、激活/退货记录、数据 | 全新 |
| 微信云开发 | 云函数(登录/激活/核销/鉴权/支付回调)+ 云数据库 + 云存储(视频/图) | 全新,是地基 |

## 三、数据模型（云数据库集合）
- **users**:`{ _openid, nickname, avatar, phone, createdAt }`
- **products**:实物商品 `{ _id, name, price, was, tag, courseId(关联课程) }`
- **courses**:`{ _id, title, chapters:[{ id, title, lessons:[{ id, name, dur, videoFileId, free }] }] }`
- **orders**:`{ _id, _openid, items, amount, status, refundable(布尔), createdAt, paidAt }`
- **qrcodes**(激活码,核心):`{ _id, code(唯一), courseId, batchId, status:'unused'|'activated', activatedBy, activatedAt }`
- **activations**(用户↔课程):`{ _id, _openid, courseId, qrcodeId, enteredAt(确认进入=退货失效点), createdAt }`

## 四、核心机制设计
1. **微信登录**:`wx.cloud.callFunction` 自动带 `_openid` → 云函数 upsert `users` → 返回用户(云开发免去自己换 code)。
2. **二维码激活**:发货前管理后台按课程批量生成 `qrcodes`(唯一 code)→ 印到实物包。用户扫**小程序码**(`scene=code`)进欢迎页 → 云函数 `activateCourse(code)`:校验未用 → 建 `activation` + 标 qrcode `activated`(防重复/防复制)。
3. **退货权节点**:欢迎页「确认进入」→ 弹窗明确告知 → 云函数 `confirmEnter`:写 `enteredAt` + 把对应 `order.refundable=false`。**有法律意义,需留时间戳证据**。
4. **课程权限**:C 端「我的课程」/目录调 `getMyCourses`(按 `_openid` 查 `activations`);播放页校验该课在用户激活集内;视频 `videoFileId` 经云函数换**临时 URL**(防盗链)。

## 安全底座（贯穿所有机制,来自云开发最佳实践）
- **不信任前端**:openid / 角色 / 价格 / 订单状态 / 激活状态,一律由云函数用 `getWXContext` + 数据库校验,不接受前端传值。
- **安全规则**:数据库 / 存储都配安全规则(替代粗粒度权限)——公共内容前端只读、用户只读写自己的、敏感集合(`orders`/`qrcodes`/`activations`)前端禁写、全走云函数。
- **云函数鉴权**:默认 `invoke: auth != null`;函数内再校验业务权限(如"这门课是不是你激活的")。
- **视频防盗链**:`videoFileId` 不直接给前端,经云函数换临时 URL,且先校验已激活。

## 五、各端功能清单
**C 端**(多数复用现有 UI,改数据源 + 加权限):登录;个人中心「我的课程」;商品/下单(接云开发订单 + 微信支付);**扫码激活欢迎页**(改造 `welcome`,加退货确认);课程目录/播放(改造 `catalog`/`player`,按权限)。
**管理后台**(全新 Web):登录;**课程内容管理**(上传视频 + 编排章节);**二维码批次**(按课程生成/导出给生产/查核销状态);订单管理(含退货处理,据 `refundable`);用户/激活记录;数据看板。

## 七、复用 / 改造 / 新建
- **复用**:`welcome`/`catalog`/`player` 与下单闭环的 UI、个人中心、现有组件、`api/request` 模式。
- **改造**:`welcome`→激活确认 + 退货弹窗;`data/course.js`→云端多课程;`store/user.js`→真微信登录;`api/`→`wx.cloud` 调用。
- **新建**:云开发后端(云函数 + 集合 + 存储)、管理后台 Web、扫码激活、权限层。

## 八、关键技术决策（待确认）
- **管理后台形态**:① 云开发自带 **CMS / 云后台**可直接管基础数据(课程/订单/用户),省大半自建;② 定制功能(**二维码批次生成/导出**、数据看板)CMS 做不了,用**云函数 + 少量自建 Web**(CloudBase Web SDK)补。推荐组合:**CMS 管数据 + 自建管定制**。
- **视频存储**:起步可用**云存储**;规模化建议**腾讯云点播 VOD**(转码 / 防盗链 / CDN)。视频大、流量贵,建议早评估 VOD。
- **微信支付**:云开发云调用支持,需**商户号 + 小程序支付资质**。
- **二维码类型**:**小程序码**(`wxacode.getUnlimited` + `scene`),扫码直达欢迎页带参。
