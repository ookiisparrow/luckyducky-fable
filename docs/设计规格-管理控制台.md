# 设计规格 · 管理控制台（课程视频 + 二维码卡片）

> 2026-06-11 立。用户需求原话：「视频上传的控制台，需要定制 UI 与上传逻辑」「要做网页版，并且逻辑清晰，支持拖拽排序，方便命名」。
> UI 设计稿：`design/console.pen`（Pencil 源文件，可在 Pencil 打开）；导出图 `design/exports/课程列表.png`、`design/exports/课程编排.png`。
> 与主规格（`设计规格-课程电商系统.md`）的关系：本文件是其 §七「管理后台自建半边」的子规格；数据模型仍以主规格 §三 为准。

## 一、定位与范围

- **给谁用**：管理员（店主 / 运营），电脑浏览器。
- **干什么**：管理课程三层结构（章节 → 课时 → 小段）＋ 给每个小段上传视频 ＋ 发布给学员。
- **对应关系**：一款材料包 = 一门课程（主规格 §四-6「持续上新」流水线的内容侧入口）；控制台编辑的就是云端 `courses` 那份数据——**引用模型**，发布后对已激活学员自动生效。
- **同时落定主规格 §七 两个待确认项**（见 关键决策记录 §14）：
  - 课程编排形态 = **自建网页编排页**（三层嵌套不进 CMS，原「seedCourses 种子当准源」过渡方案到此功成身退）；
  - 视频存储 v1 = **云开发云存储**（fileID 挂 `segment.videoFileId`，与现有数据模型一致；VOD / 防盗链留 P5 评估）。

## 二、页面与交互（与设计稿一一对应）

### 1. 课程列表页（设计稿 S1）

- 左侧栏：课程与视频（v1）／二维码卡片（v1.5，见 §六）／数据看板（留位，标「规划中」）。
- 课程卡：封面占位、标题、结构统计（N 章 · N 课时 · N 段）、**视频素材进度条（已传 / 总段数）**、状态徽标（已发布 / 草稿）、进入编排。
- 新建课程 = 先建结构再传视频；草稿课程学员不可见。

### 2. 课程编排页（设计稿 S2，核心页）

- **三层树**：章节 → 课时 → 小段，与小程序目录 / 播放器同构；章节、课时可折叠，默认只展开在编辑的那层，保持页面清爽（「逻辑清晰」）。
- **拖拽排序**：每行左侧 ⠿ 手柄，三层各自层内拖动排序；拖起的行紫描边 + 投影，落点显示紫色指示线（设计稿第 3 段即拖拽中状态）。
- **命名**：双击任意名称（课程 / 章节 / 课时 / 小段）原地变输入框，回车确认、Esc 取消（设计稿第 2 段即改名状态）；新建条目给默认名（如「第 N 段」），鼓励随手改。
- **上传**：每个小段一个上传位，三态——
  - 未上传：「拖入视频，或点击选择文件」按钮；
  - 上传中：进度条 + 百分比 + 取消；
  - 已上传：✓ 文件名 + 预览 / 替换。
  - **视频文件名不限**：归位只看所在的行，上传后自动挂到该小段（fileID 进草稿）。
- **试看开关**：对应 `segment.free`（未激活学员可看的段）。
- **保存与发布分离**：所有改动**自动保存到草稿**（顶栏显示「已自动保存 HH:mm」）；点「**发布更新**」草稿才覆盖正式内容、学员可见。顶部素材进度条提示还差几段。

### 3. 状态规则

- 草稿课程：学员不可见（小程序 `getCourses` 只读正式表）。
- 已发布课程的后续改动：存草稿不影响线上，发布时一次性生效（老学员自动看到新内容；segment 结构变更旧进度按 id 失配作废——主规格 §三 既定）。

## 三、技术路线（实现时遵循）

- **形态**：静态网页（Vue 3 + Vite 独立小工程，`admin/` 目录），部署到**云开发静态网站托管**——与小程序同一云环境、同一份数据，零跨环境同步。
- **登录与权限**（2026-06-11 实现时定稿，v1 与原方案偏差记档）：**HTTP 访问服务 + 管理口令**——网页端 fetch 调云函数 `adminApi`（HTTP 触发），口令 sha256 存 `adminConfig` 集合、每次请求校验（沿用「不信任前端」底座）；**首次登录即设置口令**（部署后立即登录占位）。选此方案的理由：零控制台配置（Web SDK 需开匿名登录 + 配安全域名）、CLI 可全程自动验证。单管理员够用；多账号/找回口令时升级 CloudBase Web SDK 账号体系，适配层已收口（`admin/src/api/cloud.js`）。
- **数据**：
  - 草稿存 `coursesDraft` 集合（与 `courses` 同形状）；编排页读写草稿。
  - 发布 = 云函数把草稿整体覆盖到 `courses` 对应文档（原子替换 + `updatedAt`）；新建课程只有发布后才进 `courses`。
  - 小程序端零改动：`getCourses` 照旧只读 `courses`。
- **上传**：Web SDK `uploadFile` 到云存储 `videos/<courseId>/<segmentId>-<时间戳>.mp4` → fileID 写进草稿对应 segment 的 `videoFileId`；「替换」= 传新文件换 fileID（旧文件清理策略记技术债，不阻塞 v1）。
- **播放衔接**：小程序播放页现为「整课一条占位视频按时长等分」；segment 真实 fileID 灌入后，播放页改为按段取 fileID 播放（主规格 §三 既定方向，属小程序侧的配套改造，单独排里程碑）。

## 四、分期

- **v1.5**：步骤⑤二维码卡片（§六，含自定义尺寸）+ 步骤⑥码批次与印刷包打包下载（SVG + 地址清单，§七）。

## 六、二维码卡片设计板块（2026-06-11 追加需求，设计稿 S3/S4）

> 用户需求原话：「产品后期会增多，二维码的卡片需要有图案设计做区分；增加一个二维码卡片页面的设计板块，能与产品二维码做绑定管理，能编辑文字、上传产品图片或插画。」

- **定位**：包装内激活卡的「卡面」设计与管理。**一款产品 / 课程 = 一张卡面**，靠图案与文字区分；卡面与该课程的全部激活码批次绑定（码批次见决策 §13）。
- **S3 卡片列表**：卡面缩略预览、绑定课程、批次统计（批次数 / 码数 / 已激活数）、状态徽标（草稿 / 已定稿）；入口「编辑卡面」「码批次」。新品上架流水线在此配卡面（与主规格 §四-6 成对上架对应）。
- **S4 卡片编辑器**：
  - 左侧画布实时预览整张卡：图案区（产品图 / 插画）→ 主标题 → 副文案 → 二维码区 → 扫码提示 → 三步说明 → 品牌条。
  - 右侧属性面板：绑定课程（下拉）；产品图 / 插画上传（PNG/JPG，可替换删除）；卡面文字三段（主标题 / 副文案 / 扫码提示，点画布元素与面板字段双向联动）；图案区底色色板。
  - **二维码区不可编辑**：按激活码自动生成（一码一张），设计器里只占位。
  - 顶栏：自动保存草稿 / 导出印刷样张（单张示例码）/ **定稿并启用**（定稿后才能生成码批次）。
- **流程**：设计卡面 → 定稿并启用 → 生成码批次 → **导出印刷文件**（一码一张卡：卡面相同、二维码各不相同，URL 按决策 §13 规则拼）→ 交付印刷厂。
- **数据**：`cards` 集合 `{ id, courseId, name, status: draft|final, art: fileID, texts: { title, sub, scanHint }, bgColor, updatedAt }`；码批次（`qrcodes.batchId`）经 courseId 与卡面关联。
- **待定（实现时拍板）**：导出印刷文件的生成方式（服务端渲染 PDF / 前端逐张合成图）；印刷尺寸与出血规格（问印刷厂要参数）。

## 七、信息架构二次细化：商品上新流水线（2026-06-11，以此为准）

> 用户明确逻辑层级（2026-06-11 两次调整后定稿）：**上传产品图片（封面图 + 其余图）→ 商品信息 → 商品 SKU → 上传编排配套教学视频 → 设计二维码卡片 → 批量生成二维码地址**。
> 调序理由（用户原话）：「我们的产品驱动是图片款式驱动」——先传图定款式，再补商品档案。

- **控制台主轴从「课程」改为「商品」**：一款商品一条流水线，六步走完即可投产（印刷 + 上架）。课程、卡片、码批次都挂在商品下（数据上经 `products.courseId` 关联，模型已支持）。
- **页面结构**（2026-06-11 三次迭代定稿：**向导式 + 双入口**）：
  - 商品列表（设计稿 S6）：每款商品一张卡，六步完成度圆点 + 状态；新建商品从第 1 步开始走向导。
  - **商品工作台 = 上新向导**（设计稿 S5）：单页，顶部六步进度条（**可点跳转**），主区一次只显示当前步内容（④ 教学视频 = S2 编排页整页嵌入、⑤ 二维码卡片 = S4 编辑器整页嵌入——大工具因此有整页空间），底部「上一步 / 下一步」推进，每步自动保存。
  - **左侧六步直达入口**（用户明确：左侧入口对应六步逻辑）：侧边栏「按步骤直达」区按流水线顺序列 ①产品图片（**封面图** 1 张——商品的「脸」，列表与详情头图；**其余图**若干，可拖排）②商品信息 ③商品 SKU ④视频上传 ⑤二维码卡片 ⑥码批次与印刷包，可**中途直切**任一步（选中商品后进对应步骤页），与向导编辑同一份数据——全流程走向导、改单项走直达，两条路都通。
  - 原 S1 课程列表 / S3 卡片列表职责分别并入 商品列表 与 二维码管理。
- **数据模型扩展**：`products` 增 `cover: fileID`（封面图）+ `images: [fileID]`（其余图，有序）——**同时喂小程序商品页**（封面图 = 列表卡与详情头图，真实素材入口）；`skus: [{ name, price }]`（与小程序详情页规格组件字段对齐，实现时细化）。
- **步骤⑤卡片编辑器增强**：**自定义卡片尺寸**（常用预设 + 自定义宽 × 高 mm；出血等印刷参数问印刷厂后定）。
- **步骤⑥批量生成与交付**：按数量生成码批次（`genQrcodes` 既有）→ **打包下载（ZIP）**：
  - `卡面.svg`——矢量设计图，由卡面数据（尺寸/图片/文案/底色/二维码占位）生成，印刷友好；
  - `二维码地址.csv`——`激活码, URL` 清单（URL 规则按决策 §13：`https://<域名>/q/<码>`）。
  - 印刷厂拿 SVG 模板 + 地址清单产卡（用户确认的工作方式：一个地址印刷一个二维码）。
- **决策 §15**：商品建档 / 图片 / SKU 进自建控制台流水线；决策 §11 中「CMS 管商品数据」相应降级为兜底查看通道。
