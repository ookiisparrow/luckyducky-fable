## 1. 正册范围

本文件是项目接口的权威登记册。这里的“接口”不只指 HTTP API，也包括：

- 小程序端 API facade：`packages/miniapp/src/api/**`
- 管理端 API facade：`packages/admin/src/api/cloud.js`
- 云函数：`packages/cloud/src/functions/**`
- `adminApi` HTTP 访问服务及其 `action`
- 支付、退款、定时任务、工作流回调
- 云存储 fileID、临时 URL、上传前缀
- DB collection 的读写契约

新增、删除、改名、改请求响应结构、改鉴权或改副作用时，必须同步更新本文件。

## 2. 接口归口规则

| 规则 | 当前正册口径 |
| --- | --- |
| 小程序页面 | 页面不直接调用云函数，优先经 `store/**`，再经 `packages/miniapp/src/api/**` 或明确的 utility facade。 |
| 管理后台 | UI 只经 `packages/admin/src/api/cloud.js`，由该文件统一 `post(action, data)` 到 `adminApi`。 |
| 云函数 | 按域放在 `packages/cloud/src/functions/<domain>/`，共享能力走 `packages/cloud/src/kit/**`。 |
| 管理 action | 统一登记在 `packages/cloud/src/functions/admin/adminApi/index.ts` 的 `ACTIONS` 查表。 |
| 支付/退款回调 | 只允许工作流服务端调用，带用户 `OPENID` 的调用视为伪造并 ACK no-op。 |
| 付费视频 | `videoFileId` 不出公开目录接口，只由 `getPlaybackUrl` 服务端鉴权后换短时 URL。 |
| 云存储 | 存储路径、读写者、是否发临时 URL 都视为接口契约。 |

扫描缺口解释：

- `cloudbase_module` 是云开发工作流平台适配入口，不是本仓库本地云函数。
- `adminApi`、`initDb`、`seedProducts`、`seedCourses`、`closeExpiredOrders` 属 HTTP/运维/定时入口，不要求有小程序端调用。
- `packages/miniapp/src/utils/cloud.js:18` 是注释中的调用示例，不是实际散落调用。
- `login` 当前由 `packages/miniapp/src/store/user.js` 直接调用云函数，`trackEvent` 由 `packages/miniapp/src/utils/track.js` 调用云函数。两者应在正册登记为特例，后续可迁到 `api/user.js`、`api/progress.js` 或保留为专用 facade。

## 11. 新增接口流程

新增接口时按这个顺序改：

1. 先确定接口种类：`clientFacade`、`cloudFunction`、`adminAction`、`webhook`、`storageContract`、`dbContract` 或 `event`。
2. 小程序端先加/改 `packages/miniapp/src/api/**`，页面不要直接 `callCloud`。
3. 管理端先加/改 `packages/admin/src/api/cloud.js` 或其后续域 facade。
4. 云端实现放到对应 domain；共享鉴权、分页、回调、工作流走 `packages/cloud/src/kit/**`。
5. 本文件登记：运行名、provider、consumer、auth、authorization、request、response、side effects、collections、storage、idempotency、privacy、tests、owner、lifecycle。
6. 高风险接口必须补测试：订单、支付、退款、激活、视频、admin、存储。
7. 若新增云函数，同步 `cloudbaserc.json`、部署闸、验证脚本和本正册。
