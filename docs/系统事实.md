## 13. 重写线（rewrite）正册

> 后端重写产物，云端唯一版本，随批部署。打包 `npm run build:rw-cloud` → `rewrite/cloud/dist/`（每函数 index.js+package.json+按需 config.json）。守卫 `rw-interface-catalog-sync` 焊本节与源码同步。

**云函数 17 个**（函数单元＝`export const main`）：

| 函数 | 现职 |
| --- | --- |
| `app` | 用户端聚合网关·30 action·公开只读五例外（`getProducts`/`getContent`/`getPageContent`/`getPublicFaq`/`getProductDetail`）无鉴权，其余 withOpenId/withRateLimit 闸 |
| `adminApi` | 管理端后台网关·HTTP 触发·115 action（源码 `ACTIONS` 112 + pre-auth `ping`/`login`/`loginByWecomCode`）·口令+会话令牌·RBAC 默认拒·认证频控·写类自动审计；部署别名 `adminApiV2`（`/adminv2`·admin 前端实际调用方，见 `scripts/lib/deploy-aliases.mjs`） |
| `payCallback` `refundCallback` | 支付/退款回调·`defineNotifyCallback` 防伪·幂等 |
| `closeExpiredOrders` `kfHealthProbe` `recallScan` `cleanupEvents` | 定时器四件：关单/客服探针/召回扫描/流水清理（`recallRules` 为库不独立成函数） |
| `kfCallback` `kfSend` | 微信客服回调（验签 fail-closed·dispatch/archive 内联）与主动发送接缝 |
| `kfMedia` | 服务端专用（`isServerCall`）：下载顾客发图临时素材（media_id 3 天有效）→ base64 回传；经 adminApi `getMediaUrl` 跨函数调用（同 `sendAgentMessage`→`kfSend` 模式）·平台接缝单点#12 |
| `genQrcodes` `seedProducts` `seedCourses` `initDb` | 激活码生成（`withAdminGate`）·种子×2·建库（清单＝COLLECTIONS 单源全量）·只部署勿调用 |
| `inspect` | 巡检机（不依赖 AI·守卫 `rw-inspect-golden`）：定时跑基建存活 + 业务不变量（db-reachable/钱守恒/卡单+死信）→写 `inspectRuns` + 每条红落 `recordAnomaly`；`isServerCall` 拒客户端·只读不改业务集合；手动触发走 adminApi `runInspect` |
| `billReconcile` | 每日拉前一日微信交易账单（`fetchTradeBill`·凭证 secureConfig/wxpay+config/pay.subMchId·守卫 `rw-bill-reconcile-golden`）→落 `wxBills`（确定性 _id 幂等）→`matchBillRows` 比对：wxOnly/金额不符→`recordAnomaly` 高危+告警（`BILL_RECON_DISCREPANCY`·指纹去重）；oursOnly 刻意不告警（留 admin `getBillMatch` 手动核）；`isServerCall` 拒客户端 |
