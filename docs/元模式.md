# 元模式——本项目治理框架的单一来源

> 立于 2026-06-13。本文是「规范框架」本身的 canonical 定义：**痛 → 本质 → 不变量 → 机器守卫 → 反向自检 → 归因** 这条流水线只在此处写一次。
> `CLAUDE.md` / hooks（`check-conventions`·`check-structure`）/ `.claude/skills/*` / `根因账本.md` / `自动化验证系统.md` **都引用本文，不再各自复述**——复述即病根#5（复制漂移）+ #11（文档渗漏）作用在框架自身描述上。
> **§A 项目无关**（可整体移植到下一个项目）；**§B 是 Lucky Ducky 实例绑定**（换项目即换）。

## §A 可移植内核（项目无关）

### A1 循环：一条痛怎么变成永不复发的机制
> 痛/主张 → 本质 → 不变量 → 机器守卫 → 反向自检 → 归因记账。

1. **痛 / 主张**：一次真实的咬人（bug / 事故 / 审核发现），或一条架构主张。
2. **本质**：抽掉表象，问「这属于哪一类结构性问题」。
3. **不变量**：一句话能判真假的事实（例：「没有任何运行时代码引 `data/`」）。
4. **机器守卫**：把不变量编进机器（见 A3 分工）。
5. **反向自检**：篡改守卫 → 必须变红 → 还原。**不变红的守卫是摆设。**
6. **归因记账**：记进病根账本；同一类的别处一并扫（顺藤摸瓜，不碎片化救火）。

### A2 信条
- **每主张 / 每病根 → 一条不变量 → 一个永久守卫。**
- **靠人记的迟早漂移，靠机器守的不会。** 能机器化的必机器化；机器化不了的，成文 + 写清「为什么必须靠人」。
- 判据：做完 = 守卫存在且绿；不回退 = 守卫永久咬；可靠 = 反向自检会红。
- **闸自己守着闸**：守卫体系的完整性本身也要被守（见 A4 末「覆盖率闸」）。
- **文档同理**：客观事实（能拿代码判真假）→ 机器维护单源、别处引用不手抄（防 stale 漂移）；主观（意图/决策）→ 单源 + provenance；**历史 → 退役**（卷档到归档层 + 原处留指针 + 索引可**唤起**·留意图/理由、退过程边角料）。
- **守卫粒度会收敛**：同一病根每实例加一条窄守卫（如 #5 漂移 / #6 占位未接真的 `*-wired` 系列）是健康起步，但属线性增长，而**守卫总数没有守卫管它**。积到查阅困难时，套已有范式 `known-collections-only`/`known-error-codes`（一张注册表 + 一条守卫，新增只加表行不加守卫）收敛——靠本信条自律，不靠机器。

### A3 守卫分工（哪种规则归哪层）
| 规则种类 | 守卫层 |
|---|---|
| 单文件样式 / 端写法 | conventions 检查（逐行） |
| 跨文件 / 仓级架构不变量 | structure 检查（repoCheck 仓级 / fileRule 逐文件） |
| 可类型表达（金额、状态名） | 类型系统（非法值编译期不可表达） |
| 行为（钱 / 权限 / 状态 / 幂等 / 分页） | 测试（业务不变量优先，反向用例锁） |
| 机器守不了（方法论 / 守则） | 写进 `CLAUDE.md`「靠人」条目 + 为什么靠人 |

### A4 五层验证（一次编辑到一次发布，层层有网）
编辑期 hook（即时）→ 提交闸（pre-commit）→ CI（push / PR）→ 反向自检（每批收尾）→ 类型系统（编译期）。
其上再加一条 **覆盖率闸**：解析「病根清单 + 守卫 provenance」，断言每条病根都有守卫或显式「靠人」豁免——把「框架是否完整」从人工对照升级为机器可证。

### A5 框架的环 = skills（每个 skill 是循环的一个常驻环节）
| 环 | 职责 | 实例 skill |
|---|---|---|
| intake 入口 | 新痛归因到病根；无匹配则**立新病根 + 配守卫** | systematic-debugging |
| genesis 改造 | 一批改动：先守卫后实现 → 反向自检 → 记账 | refactor-batch |
| health 体检 | 周期核每病根有守卫、按 P0→P3 尺子逐层验 | deep-audit |
| health·文档 | 活文档四维体检（清晰/准确/精准/简洁）·「准确」对 git/代码核 | doc-audit |
| discovery 发现 | 从执行轨迹挖「该机器化」的重复动作 | hook-audit |
| acceptance 验收 | 非技术所有者看现象验收 | acceptance-check |
| meta 自举 | 把反复用的工作流固化成 skill | writing-skills |
| **loop 总谱（串全环）** | 一整轮对抗熵增：取真值 → ②并行猎手发现(7 类熵) → 亲核 → 三选一闭环(喂回守卫/改文本/退役) → 走 genesis 落地；含跨切判断纪律(挡 over-reach/不凑数/不递扳机) | **anti-entropy-loop** |

> intake → genesis 是框架的发动机：痛进来必归因，归不进现有病根就长出新病根，框架因此**自迭代**而非一次性考古。
> **anti-entropy-loop** 不是新环，是把 ②发现(doc-audit)+落地(refactor-batch)+跨切判断**串成一圈的总谱**（conductor·composes 不 copies）。

## §B Lucky Ducky 实例绑定（换项目即换）
- **病根清单**：`docs/根因账本.md`（13 类病根·病史 → 本质 → 根治 → 落点 → 绝迹证明）。
- **守卫注册表 + 五层落地 + 四主张（T1–T4）**：`docs/自动化验证系统.md`。
- **守卫 provenance**：每条守卫带 `roots`（病根 `#N` / 主张 `TN` / 红线·基建·正册等标签）；机器守不了的病根在 `CLAUDE.md` 有「靠人」豁免锚（现为 #8 验收真实样本、#10 工具僵态）。
- **覆盖率闸**：`scripts/check-structure.mjs` 的 `guard-coverage` repoCheck（读账本 + roots + 靠人锚，把完整性变机器可证）。
- **需求→守卫闭环（P2）**：`requirement-trace` repoCheck——`需求清单` 的「需求→实现映射」每条 ✅ 实现需求有行、行内 函数(系统事实)/测试/守卫 resolve（断链当场红）+ 须守卫覆盖（钱/状态/安全类须有对应类守卫）；`npm run trace R#` 报爆炸半径。guard-coverage 的姊妹（「病根→守卫」泛化为「需求→功能→守卫」）。
- **批次推进**：`docs/现状与路线.md`（第二次重构 live 地图；第一次重构总计划 v5 见 `archive/重构总计划-v5.md`）。
- **文档退役-唤起**：`docs/archive/`（退役层）+ `docs/archive/README.md`（唤起索引）+ 守卫 `archive-index-synced`（归档⟷索引⟷活文档指针不悬空）；周期触发器 `/doc-audit`（揪活文档里该退没退的过程账）。
