## 设计宪章（四原则，每批对照）

1. **哲学——意图是资产**：先写意图（账本条目/不变量/测试）后写实现。
2. **美学——删减的艺术**：每批 commit 报净化指标；新增抽象须消灭 ≥2 处重复。
3. **结构——让错误无处藏身**：TS 非法状态不可表达；闸 fail-closed 缺省；反向用例优先。
4. **自动化——流程即守卫**：每条主张落地为「一条机器不变量 + 一个永久守卫」（check-structure 规则 / 反向测试 / 类型）；三道闸统一 `npm run check` 单一定义；做完=守卫绿、不回退=守卫永咬。详见 `自动化验证系统.md`。
