## 设计宪章（三原则，每批对照）

1. **哲学——意图是资产，代码是耗材**：先写意图（账本条目/规格/测试）后写实现；账本、总计划、CLAUDE.md、170 例回归网、console-assets 是一级交付物。
2. **美学——删减的艺术**：每批 commit 报净化指标；任何新增抽象必须以消灭 ≥2 处既有重复为代价（教训样本：已否决的账本拆分）。
3. **结构——让错误无处藏身**：TS 非法状态不可表达（状态联合类型/Fen 品牌类型）；闸 fail-closed 缺省；反向用例优先（先写「必须拒绝什么」）。

## 目标架构（B0 已证实，无需兜底）

npm workspaces 一仓四包（不用 pnpm——@dcloudio 对符号链接虚拟仓有兼容风险；npm+file: 桩机制延续）：

```
luckyducky-next/
├─ package.json            # workspaces + check 一条命令（conventions+typecheck+lint+test[+build:cloud B1起]）
├─ cloudbaserc.json        # B1 起 functionRoot 改指 packages/cloud/dist
├─ packages/
│  ├─ shared/    # TS：种子单一来源、状态联合类型/错误码、Fen 金额品牌类型；tsc 产 dist(ESM+d.ts)
│  ├─ cloud/     # TS：kit/ + functions/<fn>/index.ts + adminApi actions/ + build.mjs（esbuild bundle）
│  ├─ miniapp/   # uni-app 整体平移（已就位，JS）
│  └─ admin/     # 已平移；B7 补 eslint/vitest 基线
├─ console-assets/         # B6：八件控制台资产副本 + 库权限期望表 + drift-checklist + 重建手册
├─ scripts/                # check-conventions / guard-deploy(全拦) / deploy-fns(B5) / sensitive-fns 单源
└─ vitest.config.mjs       # test.projects: miniapp / cloud（/ admin B7）
```

**cloud-kit 内核**（账本病根 1/2/3/4/5/12 的根治载体）：`getDb`、`ok/err`、`withOpenId`、`withAdminGate`（bootstrap 须部署密钥，堵债#15）、`assertServerCall`（回调防伪内置）、`defineHttpApi`、`defineNotifyCallback`（幂等+核验 fail-closed）、`callPayFlow`（cloudbase_module 单点）、`transition`（合法流转表+钱相关自动留痕 stateEvents lite）、`ids`（确定性 _id 规范）、`paging`（统一分页协议）、`validate`（str 截断/金额数量收口）。
**打包**：esbuild 每函数单文件（external 仅 wx-server-sdk，shared/kit 内联+摇树），dist/<fn>/{index.js,package.json}，与现部署形态同构；类型检查 tsc -b 与产物 esbuild 分工。
**内存桩**：file: 桩提升根 node_modules，cloud TS 源 require 向上命中（B0 已验证解析链）。
