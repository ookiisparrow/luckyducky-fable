## 二、共存与切换策略（关键工程决策）

线上 v0.9.4 冻结运行（只止血），重写全程零干扰，靠三条规则：

1. **新函数新命名空间**：重写函数不与旧函数同名（建议少量网关化：`app`（用户端聚合网关）+ `adminApi`v2 + 独立回调），旧 38 函数原地服役；切换日前新旧并行互不可见。
2. **回调点位例外**：paynotify/refundnotify 工作流与 kfCallback 回调 URL 绑定具体函数名（控制台正册资产）——这几个点位在**切换日同名替换**（或改控制台指向），是切换 runbook 的核心步骤。
3. **数据零迁移**：新代码沿用 37 集合 + 确定性 `_id` 约定（撞键幂等），复刻上次仓切换验证过的「数据双向兼容」路数；seed/initDb 类函数继续「只部署勿调用」。
4. **测试环境先行**：新函数先对测试库（`luckyducky-test-d6fgztai8647df45`，deploy:test 通道现成）验证，再上生产命名空间。

**切换日 runbook（M5 细化）**：新版小程序提审通过 → 回调点位切换 → 发布 → 黄金路径真机验 → 48h 守望 loop → 稳定后清退旧函数与 uni-app 产物。回滚 = 旧版本回退 + 回调点位切回（next 仓完整保留）。
