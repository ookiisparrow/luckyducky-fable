## 三、微信端候选栈（A2 · 用户点名 Skyline）

### Skyline 判词（✅ 官方一手来源核实）
**没死、在更，但别把架构押在它必须赢上。**
- 官方 changelog 更到 1.4.20（2026-06-10），2022 起月更不断；底座 glass-easel GitHub 2026-07-02 仍发版；鸿蒙微信押在 Skyline 上（结构性不弃坑理由）。「暂停维护」传闻六轮检索无一手来源，判为社区体感。
- 体感的真实成分：组件库生态不适配（TDesign 无承诺、WeUI 灰度中）、社区 bug 响应慢、PC 端三年仍「规划中」（PC 上自动降级 WebView）。
- 关键机制：**不支持的客户端自动降级 WebView + 可按页开关 renderer**——官方明文推荐混用灰度路径，押注风险可降为一行配置。
- 技术约束：CSS 严格子集（无 float/Grid/em/多重阴影等，display 仅 none/flex/block）；scroll-view 需 type="list" 但换来原生吸顶/瀑布流/按需渲染；worklet 同步动画/自定义路由转场是 WebView 没有的上限；**只能自定义导航**——与本项目全页自绘导航+自绘 TabBar 天然对齐。✅

### uni-app 实锤
DCloud 无 Skyline 官方适配、战略在自研 uni-app x（uvue 原生渲染），与微信 Skyline 是两条路；透传 renderer 属野路子无人背书。继续 uni-app ≈ 与 Skyline 无缘。✅

### 坑史校准（重要预期管理）
现 9 条多端样式守卫里约一半是「微信平台坑」而非「uni-app 坑」：换原生只彻底消 2 条（rpx、uni.*/条件编译抽象）、弱化 2 条；Skyline 额外根治 sticky（原生吸顶）、可能根治遮罩穿透（⚠️ 需真机验）；新增「CSS 子集」一类新约束。**重写 ≠ 坑清零。**

### 候选排序：B > A > C
- **B（推荐基线）原生 TS + glass-easel，默认 WebView、按页开 Skyline**：拿到原生重写全部收益 + Skyline 增益按页兑现；CSS 按 Web 标准写满足「降级也正确」；Skyline 风险一行配置可逆。可先全 WebView 上线再逐页灰度，符合小批次节拍。
- **A 全量 Skyline**：体验天花板最高，但 PC 自动降级导致同一套页面双引擎验证、生态组件不适配风险全背。
- **C 保留 uni-app**：坑一条不消、编译层黑盒继续背，仅当重写目标被推翻才有意义。
- 工作量参照：23 页 / 43 组件 / 1.44 万行；无 canvas/rich-text/web-view 重包袱；store/api/utils ~2,700 行普通 JS 多数可平移；后端零波及。

## 四、后端形态与数据现实（A3）

### 锁定面（换自建后端要重做什么）
1. **openid 注入** → 自建 code2session + 自签 session。**openid 绑 AppID 不绑后端：不换 AppID 则用户身份永不丢。**✅
2. **支付/退款**（最大锁定面）：现走云开发支付连接器+工作流（凭证在控制台不在 git，8 件正册资产）。换自建 = 微信支付 APIv3 直连（下单/退款/回调验签），kit/wxpay.ts 已有 APIv3 签名原语可复用。⚠️ 风险点：商户模式——亲核 wxpay.ts 注释「自有商户号 1113881793、服务商字段普通商户留空」，直连大概率可行，**仍须 spike 验通再定案**。
3. 发货合规上传/内容安全 → 自管 access_token 直调（合规必经，工程量常规）。
4. 云存储 fileID → 换对象存储需批量迁移+改写引用；视频 CDN 是上量成本大头。
5. 定时×4、HTTP 网关、静态托管 → 常规工程量，逻辑无平台锁。
6. 微信客服/企微（wecom.ts）**已是自管 qyapi 直连**，换后端基本可平移。✅
7. 订阅消息/小程序码服务端生成：未使用，无迁移负担。✅

### 数据三档（37 集合）
- **A 真数据/钱（必须完整保全）**：users/orders/afterSales/activations/qrcodes（已印刷实物码，作废=物理损失）/progress/reviews/feedback/checkpoints/conversations/csat/wxBills/auditLog/inventory + SCM 七集合。
- **B 会话态（可重建）**：kfIdentity/csSession/agentState。
- **C 配置缓存（可弃/可重灌）**：products/courses 正草表、content/cards/config/adminConfig/kb/uploadChunks/events/rateLimit/kfState。

### 同环境共存规则（重写落地的硬约束）
- 云函数名环境级唯一：同名一部署即全量切换；真并行需改名跑影子（前端双路由代价）。
- 集合必须复用；确定性 `_id` 设计保证数据双向兼容（上次仓切换「数据零迁移」路数可复刻，archive/切换runbook.md 是现成模板）。
- seed*/initDb 只能部署勿调用（调用会重置生产数据——历史灾难根因#3）。
- 共用生产环境 = 在真数据上原地换发动机，任一部署即时影响真实用户。

## 七、SEO/GEO 与多端版图（A6）

- **小程序对站外一切搜索/AI 引擎完全不可见**；唯一例外链路 = 腾讯元宝引用公众号内容（占其信源约半）。搜一搜内优化（页面收录 sitemap/品牌官方区/服务直达）是零成本必做。✅
- **客群发现主阵地 = 小红书（手工话题 18 亿+浏览）/ 抖音（钩织播放 110 亿次）**；但小红书禁爬、不是 AI 语料源——**种草阵地与 GEO 语料阵地必须分开经营**。
- **AI 引擎信源版图**：豆包←抖音系；元宝←公众号；DeepSeek/Kimi←知乎/门户开放 web；文心←百度系。小品牌 GEO 实操 = 开放可爬官网（Q&A 结构+Schema）+ 百度百科/抖音百科 + 知乎 + 公众号。
- **web 端当代价值 = GEO 语料源 + 品牌防御，不再是搜索流量引擎**；H5 直接成交证据最弱（国内平台电商占 80–90%，无官网成交习惯）。
- **域名备案+托管两道门槛已过**（luckyducky.cn 已接 CloudBase）。SSG（如 Astro）产物可直接放现有托管，SSR 无必要。
- **三档策略候选**：案一 微信全域+静态官网名片（数天一次性）；案二 教程内容站作 GEO 燃料+小程序唯一成交口（1–2 周建站+每周 1–2 篇长期节拍，内容与小红书/公众号一鱼三吃）；案三 全渠道含 H5 商城（投入最大、证据最弱）。
- 微信生态内：小店/视频号带货大盘在涨（2025 全域带货翻倍），已有决策放开小店连接——生态内分发短期 ROI 高于站外 SEO，两者互补不互斥。
