## 2026-06-29 · 🔤 品牌字体落地（文源圆体子集 WebFont + wx.loadFontFace 远程加载·T-F1 延后项交付）
**来源**：用户问「这包里有品牌字体吗」→ 确认 handoff 带文源圆体 6 字重 OTF（84MB）。早前为快上线把品牌字体「体积过大暂缓」（uni.scss 用系统圆体栈替代·T-F1 延后），本批用子集化解掉体积阻断、落地品牌标题字。**评估（实测子集）**：源单字重 ~14MB·小程序主包仅 2MB 塞不进；子集到 GB2312 一级（3755 汉字）+ ASCII + 标点、只取设计 `--font-display` 实用的 Medium(500)/Bold(700) 两档 → woff2 M+B 合计 ~1.5MB（base64 内嵌 ×1.37·全量 6763 字 4MB——故选「常用子集 + 远程加载」）。**授权核对（抓作者 GitHub `takushun-wu/WenYuanFonts` 官方 LICENSE）**：OFL-1.1·商用免费无需署名；字体内 `fsType=0x0000`（无嵌入/子集限制）；**WebFont 特例**——仅为网页渲染子集化/格式转换（WOFF/WOFF2）、不作可安装桌面字体提供下载者**可保留保留字名** → 保留 `WenYuan Rounded SC` 不改名；基于思源黑体/宋体（同 OFL·解释名表「含他人字形」一句）。边界：能不能上是法律判断归用户/律师（[[cc-scope-mechanical-not-legal]]）·本批只陈事实+OFL 条款。**交付**：① 构建脚本 `scripts/build-brand-font.mjs`（读源 OTF→GB2312 一级子集→woff2·字符集 Node `TextDecoder('gbk')` 确定性生成·不依赖外部字表·源 OTF 不入仓）；② 子集产物入仓 `assets/brand-fonts/*.woff2`（777+792KB）+ OFL `LICENSE`（全文+版权·条件③随字体分发）+ README；③ 单源 `packages/miniapp/src/constants/brandFont.js`（family + 托管 URL + 字重描述）；④ `uni.scss` `$font-display` 前置品牌字、回退系统圆体（FOUT/子集外字优雅降级）；⑤ `App.vue` onLaunch `uni.loadFontFace({global,...})` 两字重远程加载、`fail` 不反噬启动。**关键决策**：远程加载（非 base64 内嵌）→ **包体积零增长**；产物正本在仓根 `assets/`（不在 src·不进包）。**守卫（防回退·反向自检过）**：新 `font-not-in-package`（roots 基建·锁 ① `packages/miniapp/src` 下无字体二进制(.otf/.ttf/.woff/.woff2/.eot) ② mp 可达源码无字体 data-URI base64 内嵌进 wxss）——把「14MB 字体撑爆包」永久挡门外；反向自检＝丢 woff2 进 src/static + base64 `@font-face` 进 scss → 两条各自红 → 还原绿。**产物实证（根因#8 构建≠真机）**：`build:mp-weixin` + `build:h5` 两包均**零字体二进制**；compiled `brandFont.js` 带 `url("https://www.luckyducky.cn/fonts/…woff2")`·`app.js` 带 `loadFontFace`·wxss 注入 `WenYuan Rounded SC`。**指标**：守卫 91→**92 repoCheck**（+`font-not-in-package`）·5 fileRule·测试 637→**638**（守卫 provenance 测试自动覆盖新守卫）·`npm run check` 全绿·build:mp-weixin/h5 DONE。**🔴 待靠人（你·控制台/真机）**：① 两 woff2 传 CloudBase 静态托管 `/fonts/` 下；② 小程序后台「downloadFile 合法域名」加 `www.luckyducky.cn`（否则 `loadFontFace` 真机静默失败）；③ 真机验收（woff2 能否 `loadFontFace`·Bold 真粗·FOUT 闪一下）——不行降 woff(1.89MB)/ttf(2.59MB)；④ 随 v0.9.5 republish。
