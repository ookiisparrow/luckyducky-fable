## 2026-06-24 · 🔐 P1 admin 频控加固·全局/账户级失败兜底（审核 P1·根因#13）

**痛（上线前审核 P1）**：adminApi 认证频控 key 取 `x-forwarded-for` 首个 IP（`index.ts` clientKey）——该 header **可伪造**，攻击者每次换 IP 可让 per-IP 失败计数永不达 5 次锁定阈值，**轮换伪造 header 绕过爆破锁**。原测试只覆盖「同 header 5 次失败锁定」「不同 IP 隔离」，没覆盖轮换攻击。
**不变量**：admin 认证防爆破不能只靠 per-IP；须有跨所有 IP 的**全局/账户级失败兜底**。
**修法（先守卫红→绿）**：保留 per-IP 细粒度隔离，叠加 `GLOBAL_KEY='adminlogin:global'` 全局失败计数（阈值 20/10 分·高于 per-IP 5·单管理员正常极少触发）：① `throttleGate` 查 per-IP 或全局**任一锁定即拒**（Promise.all 取较长剩余）；② `throttleFailBoth` 失败时两维度同记；③ 成功只清 per-IP，全局靠滚动窗口自然衰减（不被单次成功抹平·防分布式爆破信号丢失）。代价：全局锁期间所有 admin 登录受阻（攻击者可短时 DoS 真管理员），但 lockMs 仅 5 分自愈、控制台内部用·可接受（已注释标明）。
**新守卫 `admin-throttle-global-backstop`**（test 型 17→18·roots `#13`·reverseTest=adminThrottle.test.js）锁「轮换伪造 x-forwarded-for 仍在全局维度触顶锁定」；既有结构守卫 `admin-login-throttled`（throttleLocked/throttleFail 在场）仍绿（被 throttleGate/throttleFailBoth 内联调用）。
**净化指标**：repoCheck 仍 64 + fileRule 5、test 守卫 17→18；测试 **517→519**（adminThrottle +2：轮换 header 全局锁/全局阈高于 per-IP 不误伤别 IP）。`npm run check` 全绿、build:cloud(32) DONE（纯云函数）。**反向自检**：throttleGate 去全局只查 per-IP→轮换 header 测试红→还原绿。**仍靠人（审核留痕）**：生产验证 CloudBase 网关是否强制覆盖 `x-forwarded-for`（若覆盖则 per-IP 本就可信·全局是额外加固；未证实前 per-IP 不算强边界）——记 `待办与债`。**剩靠人**：部署 adminApi。生产仓 `2dd5a72` 零改动。
