## 优先级 backlog
1. **[真 bug · 独立] `sanitizeCart` 小数 qty**:`qty: 0.5` 能通过校验、再被 `Math.floor` 成 `0`。应要求 qty 为**整数且 ≥ 1**,补测试。(是上次 P2 修复的残留边界)
2. **[独立] `HelpSheet` 归位**:player 专属组件不该在全局 `components/`。移到 `src/pages/player/components/HelpSheet/`,并拆子面板(它过大)。
3. **[结合云开发] `SAMPLE_ADDRESS` 解耦**:`store/address.js` 从 `data/checkout.js` 引样例地址 —— 用户域依赖结算页数据。移到 `data/demo.js` 或 `data/address.js`。(云开发后地址来自云、demo 数据会废)
4. **[结合云开发] 首页 section 组件改 props 输入**:`FeatureProducts`/`Reviews`/`FAQ`/`Reassurance`/`TrustStrip` 直接 `import data/*`,同时承担展示 + 取数。改为 props,数据由页面 / adapter 提供。(云开发后数据来自云函数,adapter 本就要建,强烈建议结合)
5. **[结合云开发] 拆 `CheckoutAddonList` / `CheckoutSubmitDock`**:`checkout/index.vue` 仍承担"搭配购买"+"提交坞"两个业务块。(接云订单/支付时本就要动)
6. **[独立] 统一样式策略**:`co.scss` 与组件 scoped style 有同名样式双写,后续易漂移。公共 utility 全局化、组件样式私有化。
7. **[独立] storage constants / service**:`ld_video_intro_seen` 等协议字符串散落页面逻辑里。抽常量或 service 收敛。

主依赖方向 `pages → components / store / data / utils`,**无循环依赖**(优点)。不漂亮处即上面的
