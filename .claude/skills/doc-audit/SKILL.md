# 文档体检（②周期发现的人工节拍）

> 本 skill = 元模式 **health 环**（`docs/元模式.md` A5）的文档版，也是 loop 里 **②周期发现** 的人工触发器：①机器守卫（90+ 条）防已建模的熵复发、绿=干净；本 skill 一句话起一圈，**主动去找①扫不进来的熵**（未建模的 stale），喂回①或交人判。**它仍靠你拨**（你说「清一遍」才转），但一句话＝一整圈——这是把「手指驱动」降到最低成本，不是真自动 loop（真自动要挂 cron/`/loop`）。

审文档 = **看现象，不看声称**（A7）。曾因信了 `现状与路线` 一段旧快照、没核 git，把早被 commit 解决的 `🔴` 当成"没做"、**报低了进度**——本 skill 立身之本就是不再犯：判「准确」**必先 `git log`/代码、再看 prose；文档与 commit 打架，以 commit 为准**。

## 四维尺子（每维：机器层已有什么 / 判断层 LLM 补什么）
1. **准确（doc = 现实·最重·先做）**
   - 计数/接口/集合 → 机器层兜：`npm run check` 绿即 `*-count-synced`/`interface-catalog-sync`/`requirement-trace` 已同步。
   - 状态/进度类（`现状与路线`/`待办与债`/`重构日志`）→ 判断层：对 `git log` 最近提交 + `重构日志` 顶部条 + 「部署记账」commit **交叉核**，揪「`🔴`/占位 已被后续 commit 解决却没回写」「描述滞后于已部署代码」「`待你重部署` 其实早部署了」。
   - 法：**派生真值（git/check 输出/`系统事实`/代码）→ 比对声称 → 标矛盾**。
2. **简洁**：预算守卫已退役（2026-07-05 取消文档预算·决策§25），全靠判断层——活文档条目 >8 行（铁律①）、重复、过期未删（铁律③）、同一状态三处漂移（铁律④）、**该退役未退役的过程账**（已完成批次/已关状态/边角料还赖在活文档·核心留、过程账删正文 + 登记（git 即归档层） 史料「README.md」·见 CLAUDE 退役制）。
3. **精准**：引用不悬空（`doc-refs-exist`/`docs-pointer-liveness`/`archive-index-synced` 已守 docs 顶层 + skills 的文档指针；判断层补这些守卫扫不到的引用）；一词一义；术语口径一致。
4. **清晰**：能一眼定位、标题层级合理；客观/主观分治不混（铁律⑥）；一需求一家（铁律⑦）。

## 发现引擎（②的具体跑法·别从头手扫）

**步 0 — 先跑 `npm run check`**：机器层（count-synced / interface-catalog-sync / requirement-trace / guard-coverage / doc-refs-exist…）先过；绿了说明可机器判的计数/引用已守，本轮**聚焦机器够不到的判断层**（别重造机器已做的）。

**步 1 — 取真值基线（先于猎手·猎手要拿它核对，不然只会复读 prose）**：
```bash
# 客观计数真值（喂给猎手当对照表）
node -e "const f=require('./cloudbaserc.json').functions;console.log('云函数',Array.isArray(f)?f.length:'?')"      # 部署单元数（非 .ts 文件数！）
grep -cE "^\s+[a-zA-Z]+: '" packages/cloud/src/kit/collections.ts                                                 # 集合数（COLLECTIONS）
grep -cE "^### [0-9]" docs/根因账本.md                                                                            # 病根数
grep -n "repoCheck + .* fileRule" docs/自动化验证系统.md                                                          # 守卫计数（guard-count-synced 源）
git log -20 --oneline                                                                                             # 近批真相 + 部署记账
```
真值快照：**集合 / 云函数 / 病根 / 守卫 / 测试 各一个数 + 近 20 commit**。猎手没有它就会把「46 个 .ts 文件」当成「46 云函数」（真值 33）。

**步 2 — 并行只读猎手分组扫全部活文档**（fan-out·一次发齐并发跑）：按类分 ~4 组（参考 / 治理 / 状态 / 手册），各起一个 **Explore（只读）** 子代理，喂真值快照 + 熵地图 7 类，要求**每条 finding 给 `文件:行号 + 原文摘录 + 为什么 stale（对照真值/代码/glob）+ 建议改法`，不确定的别报、宁缺勿滥**。熵地图 7 类（要它们盯的）：
   - **E1 失活指针**：引用的某 .md 文档 / 某 archive 文档 / `§X` 不存在或已改名/归档。
   - **E2 跨文档 stale 计数**：手抄的「N 集合/云函数/守卫/病根/测试」≠ 真值。
   - **E3 预算内冗余**：≤budget 但有重复/过期/该退役的过程账。
   - **E4 archive 无界** / **E5 守卫增殖** / **E6 跨文档复述**（判断类·多为监测）。
   - **单源专项**（E2 计数 + E6 复述合看）：要答「文档单源吗」别给布尔——分三层（客观=机器证·主观=一需求一家核·prose=判断），框架见 `/anti-entropy-loop` §单源体检（不复述·单源原则本身也单源）。
   - **E7 AI-可查的通用层**：内容冗余**不因 stale/仓内重复，而因它是通用知识/官方文档已有/AI 能直接答**的东西（且常伪装成参考却自己悄悄烂）。原型＝`云开发参考.md` 184→38 行那批：砍掉 CloudBase API 教程/安全原则/官方链接×17，只留 tcb 踩坑/_openid 坑等项目特有。**判别器（必用·猎手在此最爱 over-reach）**：问「这段你会去**别处查**吗？」——**砍**＝通用 REFERENCE/教程/官方链接/API 规范（你会查官方）；**留**＝用通用语言表达的**项目 stance/方法论/决策理由/进度快照**（如 `验收手册` 四代理信号 + 根因#8 验收法、`关键决策` 的理由——你不会去「查」，那是项目的选择，**别当「软件工程 101」砍掉治理哲学**）。E7 是判断镜头、不配机器守卫（通用 vs 特有机器证不出·§7）。

**步 3 — 亲核每条 finding（验证不是相信·本 skill 的命）**：猎手会**高估严重度、误数、记错上下文**（实测：把 .ts 文件数当云函数数、把决策理由当 stale）。**逐条去代码/git/glob 核实**真值，**核过才动手**；核不实的丢弃。亲核 ≠ 信猎手。

## 改（找到 → 闭环·别只改文本）
- 真问题 → 按 `refactor-batch` 修（先核/先守卫 → 改到绿 → 反向自检 → 记账）。
- **优先看能否喂回①**：客观计数/指针类漂移，先问「是不是某守卫的扫描面漏了这文档」——能干净扩守卫面就扩（如 `collection-count-synced` 扩扫某文档、`doc-refs-exist` 范式），让它进「绿=干净」集合、不再靠手指。**②找到①该吃的，就喂给①**。
- 不能干净守的（假阳面大·如同一词在别处是子集态/历史/措辞）→ 老实改文本 + 在记账里写明「为何留给人、不上守卫」（§7 anti-overengineering：摆设守卫本身是熵）。
- 过程账退役 → archive + 留指针 + 登记 史料「README.md」（`archive-index-synced` 焊死）。

## 红线
- 判「准确」**必对机器真值（git / check 输出 / 系统事实 / 代码）核**，不靠读 prose 下结论——本 skill 全部价值在此。
- **猎手声称一律亲核**；doc 与 commit 打架，**以 commit 为准**。
- **只审计 + 按 refactor-batch 修，不破单源**——缺了就指针引单源，别复制内容到多处"补全"。
- 守 `CLAUDE.md` §4 七铁律 + 元模式 A2「客观/主观分治」。

## 产出
- 分级 findings 表（CLAUDE §7 尺子）：P0 误导性失真（读了会误判）｜P1 stale 过期｜P2 冗余/越界/漂移｜P3 风格；每条带 **「文档 : 证据 : 真值来源」** + 已修/喂回守卫/留给人。
- 业务语言汇报；够不到的（需所有者确认）老实标。

> 一句话：一句话起一圈——先 check 过机器层、取真值快照、并行猎手扫全部活文档找①扫不进的熵、**每条亲核**、能喂回守卫就喂、不能的老实改文本并写明、过程账退役，全交 refactor-batch 落地。
