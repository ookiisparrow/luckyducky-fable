# 全仓体检（打包扳机）

> 体检家族的唯一入口（2026-07-23 拍板重构：原 anti-entropy-loop/patrol 并入本扳机退役，两个单元 /doc-audit、/deep-audit 保留可单独调）。**仍是人工/循环节拍**：你说「体检/治熵」转一整圈；挂 `/loop /audit 轻量` 即周期巡检。

## 两档

- **全量档**（默认·「全面体检/治熵/清一遍」）：两面齐跑，可并行 fan-out。
- **轻量档**（「/loop /audit 轻量」或「巡逻」）：每轮只跑一面、轮换（上轮文档面→本轮代码面），findings 直接落 `docs/调试日志.md`/`docs/待办与债.md`——无独立轮报账本。

## 一圈五步

1. **停轮闸 + 基线**：`git status --short` 非空 → 停轮上报（不卷别人半成品）；跑 `npm run check`——机器层绿=已建模的熵干净，本圈只找机器够不到的。
2. **文档面**：走 `/doc-audit`（真值快照 → 并行猎手扫活文档 → 四维尺子 + 7 类熵镜头）。
3. **代码面**：走 `/deep-audit`（P0→P3 逐层验交易链/安全闸/状态机 + 核每病根有守卫）。
4. **亲核每条（验证不是相信·本扳机的命）**：猎手/子代理会误数、高估、记错——逐条对代码/git 核实，核过才动手；核不实的丢弃。
5. **三选一闭环（别只改文本）**：
   - **喂回守卫**：客观可机器判的漂移（计数/指针/tally）→ 扩/建守卫，让这类熵绝迹；
   - **改文本 + 写明为何留人**：判断题/假阳面大 → 改文本，记账写清「为何不上守卫」（不摆设守卫）；
   - **退役**：已闭过程账 → 登记 `docs/史料索引.md` + 原处留指针 + 删正文（git 即归档层）。
   落地全走 `/refactor-batch`（守卫先红 → 改到绿 → 反向自检 → 记账）。

## 跨切判断纪律（历轮攒下·两面通用）

- **验证不是相信**：doc 与 commit 打架，以 commit 为准。
- **挡 over-reach**：猎手爱把「项目 stance/方法论/决策理由」当 stale 砍。判别器——「这段你会去**别处查**吗？」会 → 砍通用 reference；不会 → 留（治理哲学不当「软件工程 101」砍）。
- **不为凑数干活**：一圈无真问题 → 报「跑了·验了·干净」并停，负结果也是交付。
- **不递扳机**：well-founded 的判断自己拍；只在真分叉（动钱/不可逆/动用户资产/口径未定）才问用户。
- **不破单源**：缺了就指针引单源，别复制内容到多处「补全」。

## 红线

- 不部署、不动云端（数据库/控制台/工作流）——共用真实生产环境，体检只在仓内。
- 亲核 > 相信；落地不绕 `npm run check`；反向自检不变红的守卫=摆设。

> 一句话：一声令下转一整圈——停轮闸 → 基线 → 文档面+代码面 → 每条亲核 → 三选一闭环 → refactor-batch 落地；轻量档单面轮换可挂 /loop。
