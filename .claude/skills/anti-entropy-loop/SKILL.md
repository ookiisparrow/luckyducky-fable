# 治熵一圈（anti-entropy loop · 执行动作总谱）

> 「一整轮对抗熵增」的总谱——把散在 `元模式.md`(loop 骨架) / `/doc-audit`(②发现) / `/refactor-batch`(落地) 里的环，加上一程攒下的**跨切判断纪律**，串成一条可一句话起的圈。**composes 不 copies**：子步细节引用对应 skill、本文不复述（破复述即病根#5/#11）。**仍是人工节拍**（你说「治熵/清一遍」才转一圈），非自动 cron——真自动挂 `/loop` 是另一档。

## loop 架构（一句话）
- **① 守卫（防）**：机器守卫（本仓 90+ 条）拦已建模的熵·绿=干净·零成本常驻。
- **② 发现（找）**：主动找①扫不进来的**未建模**熵，喂回①或交人判。
- **人（判残余）**：只判机器证不出的；**自己能拍的就拍**（别把判断全递回用户=另一种 finger-driven）。
- **治的熵**（解决的问题类）：失活指针 / 跨文档 stale 计数 / 过期状态没回写 / 该退役的过程账没退 / **AI-可查的通用层** / 守卫扫描面盲区——文档与治理层「只增不减」的熵。

## 一圈五步
1. **取真值快照**（先于猎手·否则它们只复读 prose）：跑 `npm run check` 过机器层 → 取客观计数真值（**云函数 = `cloudbaserc.json` functions 数·不是 .ts 文件数**／集合 = COLLECTIONS／病根／守卫／测试）+ `git log -20`。命令清单见 `/doc-audit` 步1。
2. **发现**：并行只读猎手（Explore）分组扫全活文档，喂真值快照 + **7 类熵镜头 E1–E7**（E1 失活指针 / E2 跨文档 stale 计数 / E3 预算内冗余 / E4 archive 无界 / E5 守卫增殖 / E6 跨文档复述 / **E7 AI-可查通用层**）。引擎细节走 `/doc-audit` 步2。
3. **亲核每条（验证不是相信·本 loop 的命）**：猎手会误数 / 高估 / 记错上下文——逐条去代码 / git / glob 核，**核过才动手**。实测救场：把 .ts 数当云函数（46≠33）、需求 tally（28≠29）、把治理 stance 当「101」要砍。
4. **闭环决策（三选一·别只改文本）**：
   - **喂回①**：客观可机器导的漂移（计数 / 指针 / tally）→ 扩 / 建守卫（`doc-refs-exist`／`requirement-tally-synced`／`collection-count-synced` 范式），让这类熵绝迹、不再靠手指。
   - **改文本 + 写明为何留人**：判断题 / 假阳面大（通用 vs 特有、过度抽象）→ 改文本，记账写清「为何不上守卫」。**不摆设守卫**（§7·摆设守卫本身是熵）。
   - **退役**：已闭过程账 → archive + 留指针 + 登记索引（`archive-index-synced` 焊）。
5. **落地**：走 `/refactor-batch`（根因 → 守卫先红 → 改到绿 → **反向自检** → `npm run check` 全绿 → 记 `重构日志`）。**git 落地按环境分叉**：本机 = 直接上 main（不走 PR·refactor-batch §收尾）；**远程沙箱 = 开 `claude/xxx` 分支 → PR → 等 CI 绿 → squash 合并 → 拉回 main**（harness 规则·不直接 push main）。

## 跨切判断纪律（一程攒下·别再只活在脑子里）
- **验证不是相信**：猎手 / agent / 自己的记忆一律亲核对代码 / git；doc 与 commit 打架，以 commit 为准。
- **挡 over-reach（猎手最常犯）**：它爱把「项目 stance / 方法论 / 决策理由」当 stale / 101 砍。判别器——「这段你会去**别处查**吗？会 → 砍通用 reference；不会 → 留」。`验收手册` 四代理信号、`根因账本` 病根、`关键决策` 理由：**绝不当「软件工程 101」砍掉治理哲学**。
- **不为凑数干活**：一圈扫完若无真问题，报「跑了·验了·干净」并停——别硬砍一篇凑业绩（违判别器）。**负结果也是交付**。
- **不递扳机**：well-founded 的判断自己拍、错了叫停；只在**真分叉**（动钱 / 不可逆 / 动用户资产 / 口径未定）才 `AskUserQuestion`。把「该不该」全推回用户 = finger-driven。
- **不破单源**：缺了指针引单源，别复制内容到多处「补全」；计数 / 事实指机器单源、不手抄。

## 这一程解决过的问题（锚·防复议·亦本 skill 的来历）
active-15 之外文档层盲区 → `doc-refs-exist`；待办闭账审计批退役 archive；需求复核 tally 漂 → `requirement-tally-synced`；`云开发参考` AI-可查通用层 184→38 行；立 **E7** 第 7 熵类喂回 `/doc-audit`；把整套执行动作固化成本 skill（终结「执行动作只活在对话里」这条 finger-driven）。净：守卫 89→91·测试 636→637·活文档 15 瘦身。

## 红线
- 亲核 > 相信；落地不绕 `npm run check`；反向自检不变红的守卫 = 摆设。
- 不摆设守卫（判断题不上守卫）·不为凑数砍·不破单源·不把判断全递回用户。
- 远程沙箱走 PR → CI → 合并（不直接 push main）；本机走直接上 main——别混。
- 纯文档 / 治理改不碰运行时；碰运行时按 `/refactor-batch` 验 build。

> 一句话：一句话起一圈——取真值 → 并行猎手扫 7 类熵 → **每条亲核** → 三选一闭环（喂回守卫 / 改文本写明 / 退役）→ 走 `/refactor-batch` 落地（沙箱 PR·本机直上 main）；挡 over-reach、不凑数、不递扳机、不破单源。
