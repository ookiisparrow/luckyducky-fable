## 上下文一句话
Lucky Ducky 重写线（fable）的 admin 后台**重画**——旧文件两代叠加太乱，用户拍板新开干净文件从零一屏屏画。整体战役计划见 `~/.claude/plans/doc-claude-md-web-admin-functional-ember.md`（本项是其中 M-adminDesign）。

## 画法纪律（务必遵守，否则又会乱）
- **一屏 = 一块独立画板**，侧栏**直接画在屏里**（不抽公共组件、不在旁边放组件区——用户明确嫌"混在一起"）。
- 屏竖排在 **x:0**，每屏 **1440×940**，**y 间隔 100**（下一屏 y = 上一屏 y+1040）。
- 屏 `fill:$bg-grey`、`layout:horizontal`、`clip:true`；左侧栏 248 宽白底，右 Main `fill_container`。
- 每屏画完 `get_screenshot` 核一眼再下一张。

## 侧栏导航（6 顶层·用户在 HTML 工具里定稿·别再改结构）
1. **数据看板**（item, chart-column）
2. **商品与上新**（item, tag）→ 点进 = 六步上新向导（工作流·侧栏切「按步直达」聚焦态）
3. **订单履约**（group, package）→ 订单发货 / 售后退款 / 财务对账
4. **客服 360**（group, headset）→ 会话 / 客户 360 / 知识库 / 满意度 / 坐席
5. **进销存**（item, factory）→ 点进 = 进销存工作流（同向导，工作流·按步直达；步骤顺序未定，画时先拟一版给用户改）
6. **系统**（group, settings）→ 异常 / 巡检 / 外部后台 / 消息通知 / 设置

侧栏本身要支持「拖拽排序 + 父子嵌套」（运行时可配置·全局·超管配一次）——设计上有「编辑排序」入口；实现活已记进任务 M-adminImpl。

### 侧栏 batch_design 骨架（每屏复制这套 helper，改 active/expanded）
Logo（🦆 badge + Lucky Ducky/运营控制台）→ 「导航」caption+编辑排序 chip → item/group/child 行 → Spacer(fill_container) → Admin(小天/管理员) 页脚。
- item：`padding:[9,10,9,12]`，active 时 `fill:$bg-lilac`+`stroke:$purple-line`、icon `$brand`、label `$purple-ink`600。
- group header：caret(chevron-right/down)+icon+label(600 $ink)。
- child：`padding:[7,12,7,42]`+dot(5×5)，active 同上。
- 展开的组：header caret=chevron-down，其后紧跟 child 行。
- 向导/工作流态侧栏：换成「返回运营台」+ 商品与上新(active 头) + 「按步骤直达」+ 编号步骤行（当前步 brand 数字徽章高亮）。
