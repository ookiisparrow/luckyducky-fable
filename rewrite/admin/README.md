# rewrite/admin · 管理控制台 v2（M3）走查手册

> **为什么靠人**（根因#8）：后端 action 全集（单源 `rewrite/cloud/src/functions/adminApi/index.ts` ACTIONS 注册表·计数由守卫焊在 `docs/系统事实.md`，此处不手抄）有全量测试盯着，但「真登录真操作顺不顺手」只有人验得了。本手册 = M3 验收的人工半边。
>
> **本机启动**：`cd rewrite/admin && npm run dev` → 浏览器开提示的地址 → 用管理口令登录。
> **在线部署**：`npm run build` 产物传静态托管**正式路径 `/admin`**（2026-07-09 M5 切换起为线上唯一版本·并行期 `/admin2` 已退役）；部署前先清空旧目录防哈希堆积（`tcb hosting delete admin --dir --yes` 再 deploy·坑见 `docs/云开发参考.md`）。部署是用户动作。

## 运营九类动作演练清单（照做一遍 ✅/❌）

| # | 动作 | 在哪 | 演练什么 | 结果 |
|---|---|---|---|---|
| 1 | 看经营 | 总览·数据看板 | 五卡数字对不对（订单数/成交额与旧台一致）、钱链异常区 | |
| 2 | 上新 | 商品与内容·商品管理 | 新建商品→填核心字段→传封面→加规格→上架；缺封面上架被人话拦 | |
| 3 | 上下架/橱窗 | 商品管理 + 小程序橱窗 | 下架两步确认→小程序里消失；恢复销售→回来；改排序小程序立即生效 | |
| 4 | 课程与视频 | 课程管理 | 载入 course-xxx→改段名→传一段视频（看进度条）→保存草稿→发布两步确认 | |
| 5 | 发货 | 订单与钱·订单发货 | 待发货单填快递单号发货；金额不符单禁发标红 | |
| 6 | 审退款 | 售后退款 | 同意两步确认（第二步显金额）；拒绝必填原因；**失败时错误码原样可见** | |
| 7 | 对账 | 财务对账 | 累计三卡、按日分桶；拉某日账单→四类勾对刷新 | |
| 8 | 客服督导 | 客服组五页 | 搜客户看 360（单面板挂不全黑）、查会话时间轴、改 FAQ、看满意度 | |
| 9 | 管账号/码/库存 | 系统组 + 库存 | 建外包号→停用两步确认；生成激活码批次→复制码；改库存（刚被下单占用会拒并说明） | |

**横切**：随手点错东西看错误提示是不是人话；退出登录再进；两个标签页同时开不串。
