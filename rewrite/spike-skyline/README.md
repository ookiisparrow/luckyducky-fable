# Skyline 真机 spike（M0 批4 · 用完即弃）

> 目的：给「按页开 Skyline」的决策补上真机证据（根因#8：构建过 ≠ 真机能用）。两页**内容完全相同**，一页 Skyline 渲染、一页 WebView 渲染，逐项对比五张测试卡。结论回填本文件底部记录表 → 记入 `docs/重构日志.md`，决定 M2 哪些页首发开 Skyline。
> spike 定性：**验证完即弃**，不进新线地基（真 app 在 `rewrite/mp/`，M2 立包，用 TS；本工程 JS 图省事）。

## 前置版本

- 微信开发者工具 ≥ 1.06.2307260（推荐最新 Stable/Nightly）
- 调试基础库 ≥ 3.0.2（工具右上角「详情→本地设置→调试基础库」，本工程预设 3.4.3）
- 手机微信 ≥ 8.0.40（T4 的 position:fixed 需 ≥ 8.0.43）

## 操作步骤

1. 微信开发者工具 →「导入项目」→ 目录选 `rewrite/spike-skyline/`（AppID 已预填本店小程序）。
2. 工具内先跑一遍：入口页两个按钮分别进「Skyline 页」「WebView 页」，**看顶栏右侧徽标**——显示 `skyline` / `webview` 即渲染引擎正确（版本不满足时 Skyline 会静默降级 WebView，以徽标为准，别猜）。
3. 点「预览」扫码 → **真机**上重复：两页各过一遍五张卡，按下方记录表逐项记 ✅/❌/现象。
4. 有异常的项顺手截图。测完把记录表结果发我即可（拍照/打字都行）。

## 记录表（真机为准；工具内表现仅参考）

| 测试项 | 看什么 | Skyline 页 | WebView 页 |
|---|---|---|---|
| 徽标 renderer | 顶栏徽标是否分别为 skyline / webview | ✅ | ✅ |
| T1 CSS 变量 | 左块紫=支持 var()；右块橙=支持行内注入变量；灰=不支持 | ✅ | ✅ |
| T2 遮罩穿透 | 遮罩上滑动，背景列表动不动（不动=✅） | ✅ 已锁 | ✅ 已锁 |
| T3 SVG | 左图紫→黄渐变正常显示；右图纯紫参照 | ✅ | ✅ |
| T4 悬浮 TabBar | 底部紫色药丸贴底悬浮、滚动时不动、不被安全区吃掉 | ✅ | ✅ |
| T5 吸顶 | Skyline 页黄条应吸顶（原生 sticky-header）；WebView 页预期不吸（复现旧坑） | ❌ 未吸 | ❌ 未吸（符合旧坑预期） |
| 整体 | 滚动手感/字体渲染/布局有无明显差异 | 正常 | 正常 |

> **已回填（2026-07-04 用户真机实测）**。T5 Skyline 未吸顶疑因 scroll-view 内混排普通节点与 sticky-section——吸顶增益按「未证实」记账，M2 目录页用纯 sticky-section 结构复验；其余结论已记 `docs/重构日志.md` M0 批5。

## 结论去向

- 三个存疑点（T1 变量 / T2 穿透 / T3 SVG）的真机结论 → `docs/重构日志.md` M0 批4 记账，M2 按页开 Skyline 的名单据此定。
- 出现「Skyline 页布局明显崩」＝ CSS 子集踩线，记下具体卡片，重写时该模式列入新线守卫。
