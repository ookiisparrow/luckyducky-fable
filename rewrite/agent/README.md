# rewrite/agent · 坐席台 v2（M3）走查手册

> **为什么靠人**（根因#8）：轮询合并、认领互斥都有测试钉着，但「接待顺不顺手、消息会不会闪重」只有真用验得了。**坐席台是在用人工客服唯一通道——切换必须零断档**：旧台照跑，新台并行验，验过才在 M5 切路径。
>
> **本机启动**：`cd rewrite/agent && npm run dev`（端口 5174）→ 坐席口令登录（或企微内带 ?code= 免登）。

## 接待全流程演练（需要两个身份配合：你的手机扮顾客 + 浏览器扮坐席）

| # | 步骤 | 看什么 | 结果 |
|---|---|---|---|
| 1 | 顾客在小程序找客服发一句话 | 坐席台 10 秒内队列出现该会话、显示「等了 X 秒」 | |
| 2 | 点「接」 | 会话进「我在接」、聊天窗打开、能看到顾客点转人工**之前**说的话（10 分钟回溯） | |
| 3 | 回复顾客 | 顾客微信里收到；**坐席输入框只在发送成功后清空** | |
| 4 | 顾客再发消息 | 3 秒内出现在聊天窗、**不出现重复气泡** | |
| 5 | 看右侧 360 | 顾客同意过数据共享→显示画像/订单；没同意→如实提示「按规定不能看」 | |
| 6 | 第二个坐席账号同时点「接」同一单 | 后到者被拒「已被别人接走」 | |
| 7 | 放回队列 / 升级给商户 | 会话回队列可再接；升级件在**超管**队列里标黄可重接 | |
| 8 | 结束会话 | 顾客收到满意度提示「回复 1-5 分」；顾客回 3 → 满意度报表 +1 条 | |
| 9 | 切状态 在线→忙碌→离线 | 不报错（推送分配随状态·后台行为） | |
| 10 | 关掉标签页再开 | 「我在接」还在（刷新不丢现场） | |

## 与旧台并行的红线
- 演练期间旧坐席台照常用；两台操作同一批会话数据完全兼容（同一个后端状态机）。
- M5 前**不**下线旧台；新台验收通过 + 你拍板后才切静态托管路径。
