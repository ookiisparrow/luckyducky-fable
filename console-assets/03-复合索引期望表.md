# 03 · 复合索引期望表（根因#9 · 容量前置 2026-07-12）

> **这是什么**：`rewrite/cloud` 全部「where + orderBy」复合查询形状的**建索引靶**。云开发数据库在数据量上去后，复合查询没有匹配索引会退化全表扫描（慢/超时）。索引建在控制台、git 管不到——与 `02-库权限期望表.md` 同性质的 git 外资产正册。
> **现状（2026-07-12 立册时）**：生产库现有索引**未逐一核对**（无真实用户、量小，尚不构成瓶颈）；本表为「量上来前照单建」的期望态。**变更纪律同 README：先改本表、再动控制台。**
> **怎么建**：云开发控制台 → 数据库 → 选集合 → 索引管理 → 新建复合索引，字段与方向照下表。控制台地址：https://tcb.cloud.tencent.com/dev
> **方向说明**：MongoDB 系复合索引支持整体反向遍历（(a↑,b↓) 同时服务 (a↓,b↑)），且前缀字段为等值过滤时末位排序字段正反都可用——下表已按此合并同型条目；出处 file:line 为 2026-07-12 快照，代码改形态时同批更新本表。
