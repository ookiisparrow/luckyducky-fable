## 资产细节

### ① 连接器 wxpay_33nb7su
- 凭证：**APIv3 密钥** + **API 证书**（商户平台下载）。**REDACTED**——重建时从商户平台重取。
- ⚠️ 踩坑：**证书序列号不能带空格**（粘贴时易混入，导致签名失败）。

### ② 主支付工作流 sywxzfapifqzf_nncvqss2
- 方法：JSAPI 下单（cloudbase_module 触发，flowId 存 config.pay.flowId）。
- **入参表达式注入**：`notify_url`（指向 paynotify）+ `payer.openid`（由 pay 云函数显式传入，不依赖工作流环境注入）。
- 入参由云函数 `pay` 经 `kit.callFlow` 传：`{ description, out_trade_no, amount:{total,currency}, payer:{openid} }`（金额取库内订单，不信前端）。
- 验证终点信号：配到「openid 必填」即签名/回调地址全通。

### ③ 退款工作流 kbgzl_n8gojr3a
- 方法=**申请退款**，数据源=连接器 wxpay_33nb7su，`notify_url`=refundnotify（尾号 1863425）。
- 触发：`adminApi.approveRefund` 经 `kit.callFlow` 传 `{ out_trade_no, out_refund_no, reason, amount:{refund,total,currency} }`（金额取售后单云端分摊额）。
- 收尾：refundnotify → 转发节点 → refundCallback 幂等置 refunded。

### ④⑤ 回调流 + ⑥ 转发节点
- paynotify/refundnotify 收微信通知后，**script 节点转发**给 payCallback/refundCallback（代码见 `forward-node.js`）。
- 转发节点用 `@cloudbase/node-sdk` 的 callFunction，data=`wepayTrigger.output.resource||{}`（空数据防御）。
- **paynotify 两条同码双保险**（幂等安全）；正册=URL 含 `2065345255437447170`。
- ⚠️ **工作流改完必须「发布」**才能被云函数调用（运维坑，见 README 坑册）。

### ⑦ config 集合 pay 文档（代码↔工作流接缝，**在云数据库，可 tcb 读**）
```json
{
  "_id": "pay",
  "mode": "real",
  "subMchId": "1113881793",
  "flowId": "sywxzfapifqzf_nncvqss2",
  "refundFlowId": "kbgzl_n8gojr3a"
}
```
- `mode`：`mock`（开发期模拟支付，createOrder 直接 paid）/ `real`（真实支付，已切 real）。
- createOrder 读 mode 决定 pending/paid；pay 读 flowId 触发工作流；approveRefund 读 refundFlowId。
