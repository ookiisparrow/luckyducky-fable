# 数据库 / 云存储 权限期望表（根因账本 #9，固化验收单 X）

> 「敏感写一律走云函数」的前提是**客户端不能直接写库**。这道闸在云开发控制台设置、**代码看不到**（CLI 也读不到安全规则）——故是 git 外资产，须人工核对。本表是期望档位的正册副本（生产仓验收单 X 固化），上线前/迁移后逐项核一次。
>
> 怎么核：微信开发者工具 → 云开发 → 数据库 → 逐集合 → 右上「权限设置」看当前档位。**绝对不能有集合是「所有人可读写」**。

## 16 个集合期望档位

| 集合 | 期望档位 | 写入口 |
|---|---|---|
| `users` | 仅创建者可读写 / 仅后台 | login / updateProfile |
| `orders` | 仅创建者 / 仅后台 | createOrder / shipOrder / confirmReceive / confirmEnter |
| `qrcodes` | 仅后台（客户端禁写） | genQrcodes / activateCourse |
| `activations` | 仅创建者 / 仅后台 | activateCourse / confirmEnter |
| `progress` | 仅创建者 / 仅后台 | trackEvent |
| `events` | 仅创建者 / 仅后台 | trackEvent |
| `reviews` | 仅创建者 / 仅后台 | submitReview |
| `afterSales` | 仅创建者 / 仅后台 | applyRefund / refundCallback / adminApi |
| `adminConfig` | 仅后台（口令/设置） | adminApi |
| `productsDraft` | 仅后台 | adminApi.saveDraft |
| `coursesDraft` | 仅后台 | adminApi.saveCourseDraft |
| `cards` | 仅后台 | adminApi.saveCard |
| `uploadChunks` | 仅后台（临时中转） | adminApi.uploadChunk |
| `config` | 仅后台（支付开关/flowId） | 控制台/CLI |
| `products` | **所有人可读、仅后台可写** | adminApi.publishProduct |
| `courses` | **所有人可读、仅后台可写** | adminApi.publishCourse / seedCourses |
| `content` | **所有人可读、仅后台可写** | adminApi.saveHomeContent |

> 公开读三集合（products/courses/content）是商店展示内容，本就要所有人可见；其余一律严格。
> 发现任一集合是「所有人可读写」→ 改「仅创建者可读写」→ 记录核对（验收单 X 第 3 条）。

## 云存储

- `products/` `videos/` `avatars/`：**仅创建者可写、所有人可读**（图片/视频要能展示，但不能被乱传/篡改）。

## 备注：debt #14 与确定性 _id

- 写侧确定性 _id（submitReview `orderId__productId`、afterSales `orderId__productId`、qrcodes/code、seed `_id`=业务 id）让「仅创建者可读写」即够（撞 _id 抛错=天然唯一）。
- users/progress 仍「先查后建」（债#14），其确定性 _id 惰性迁移完成后，权限模型不变（仍仅创建者/仅后台）。
